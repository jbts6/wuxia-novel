# 多书籍支持 + 全局搜索功能规格说明

## 背景

当前可视化仪表盘只支持查看一本书（《多情剑客无情剑》），用户希望能够在同一界面中切换查看仓库内所有已解构的小说（共17本）。

## 目标

1. 支持多书籍切换查看
2. 书籍选择器使用二级菜单（作者→书名）
3. 实现全局搜索功能
4. 记住用户最近查看的书籍

## 功能需求

### 1. 书籍选择器

- **位置**：页面顶部导航栏左侧
- **形式**：下拉菜单，一级为作者，二级为书名
- **显示信息**：作者名、书名、角色数量
- **交互**：点击书名切换当前查看的书籍
- **状态**：当前选中的书籍高亮显示

### 2. 全局搜索

- **位置**：页面顶部导航栏右侧
- **形式**：搜索输入框 + Dropdown结果列表
- **搜索范围**：角色名、技能名、物品名、事件名
- **搜索逻辑**：模糊匹配，输入关键词实时搜索
- **结果展示**：
  - 按类型分组（角色、技能、物品、事件）
  - 每个结果显示名称、类型标签、一句话描述
  - 点击结果打开对应的详情面板
- **空结果**：显示"未找到匹配结果"

### 3. 加载状态

- **切换书籍时**：显示全屏Spin加载动画
- **加载提示**：显示"正在加载《书名》..."
- **错误处理**：加载失败时显示错误提示

### 4. 数据持久化

- **存储位置**：localStorage
- **存储键**：`novel-dashboard-last-book`
- **存储值**：书籍路径字符串（如"金庸/天龙八部"）
- **读取时机**：页面初始化时读取，作为默认选中书籍
- **写入时机**：用户切换书籍时写入

## 数据架构

### 书籍元数据格式 (`public/data/books.json`)

```json
[
  {
    "path": "古龙/多情剑客无情剑",
    "author": "古龙",
    "name": "多情剑客无情剑",
    "characters": 32
  },
  {
    "path": "金庸/天龙八部",
    "author": "金庸", 
    "name": "天龙八部",
    "characters": 210
  }
]
```

### 数据加载方式

- **方案**：Vite插件读取本地文件系统
- **API路径**：`/api/novel/{书名}/{文件名}`
- **参数**：`?book={书籍路径}`

## 技术实现

### 状态管理

- **useBookStore**：管理书籍列表、当前选中书籍、加载状态
- **useNovelStore**：修改为接受动态bookPath参数

### 组件结构

```

```
