# 武侠知识库 Dashboard 重做计划

## 目标

从头构建全新的 dashboard，适配 generate-kb 生成的新知识库结构。

## 用户选择

- **设计风格**：全新设计（不复用水墨宣纸风格）
- **数据加载**：静态数据打包（构建时打包 JSON，不依赖 API 服务器）
- **页面范围**：9 个页面（Library, BookOverview, Characters, Skills, Items, Factions, Locations, ChapterSummaries, Dialogues）
- **UI 库**：shadcn/ui + Tailwind CSS（不用 Ant Design）

---

## 技术栈

- React 18 + TypeScript
- Vite 构建
- **shadcn/ui**：复制粘贴组件，完全可控，无依赖锁定
- **Tailwind CSS**：原子化 CSS，设计自由度高
- **Radix UI**：shadcn/ui 的底层原语（自动引入）
- Zustand（状态管理）
- React Router 6（路由）
- **TanStack Table**：高性能表格（替代 antd Table）
- 直接 `import` JSON 文件（Vite 原生支持，tree-shakeable）

## UI 设计方向

**温暖典雅 + 数据密集** 风格，使用 shadcn/ui 组件系统。

### 设计原则
1. **清晰优先**：信息密度高，但不杂乱
2. **现代布局**：卡片式、响应式
3. **古典点缀**：在关键位置使用古典元素（如印章图标、书卷纹理），但不主导整体风格
4. **shadcn/ui 风格**：圆角、柔和阴影、中性色调

### 设计系统（基于 ui-ux-pro-max 推荐）

**风格**：Data-Dense Dashboard
- 支持亮色/暗色模式
- 关键词：数据表格、KPI 卡片、网格布局、空间高效

**配色方案**（暖色系，呼应武侠主题）：

| 用途 | 色值 | Tailwind 变量 |
|------|------|---------------|
| Primary | `#78716C` | `--primary` (暖墨灰) |
| Accent | `#D97706` | `--accent` (琥珀) |
| Background | `#FFFBEB` | `--background` (奶油白) |
| Foreground | `#0F172A` | `--foreground` (深墨) |

## 页面设计

### 1. Library（全库总览）
- 书籍卡片网格，按作者分组
- 每张卡片显示：书名、作者、统计数字（角色数、技能数等）
- 点击进入书籍

### 2. BookOverview（书籍概览）
- 统计摘要：角色/技能/物品/势力/地点数量
- 核心角色列表（前 10）
- 重要势力列表
- 章节时间线（可选）

### 3. Characters（人物志）
- 表格列表 + 筛选器（按 role, archetype, power_rank, faction）
- 列：名称、别名、身份、门派、境界、简介
- 点击打开详情面板
- 详情面板：基础信息、性格特征、关系网络、武功列表、经典台词

### 4. Skills（武功阁）
- 表格列表 + 筛选器（按 type, faction, mastery_rank）
- 列：名称、类型、门派、境界、简介
- 详情面板：基础信息、招式列表、战斗风格、掌握人物

### 5. Items（兵器谱）
- 表格列表 + 筛选器（按 type, rarity_tier, tags）
- 列：名称、类型、标签、稀有度、简介
- 详情面板：基础信息、描述、效果、关联技能、关联角色

### 6. Factions（势力录）
- 表格列表 + 筛选器（按 type）
- 列：名称、类型、地点、领袖、简介
- 详情面板：基础信息、下属机构、成员列表

### 7. Locations（地点志）
- 表格列表 + 筛选器（按 region）
- 列：名称、区域、关联势力、简介
- 详情面板：基础信息、关联势力、关联角色

### 8. ChapterSummaries（章回录）
- 时间线/卡片列表
- 每章卡片：章节标题、摘要、关键事件、关键角色
- 点击角色可跳转到角色详情

### 9. Dialogues（对话集）
- 列表 + 筛选器（按 chapter, speaker, tone）
- 对话气泡样式（类似 IM）
- 显示：说话者、语气、章节、原文

## 风险和注意事项

1. **JSON 文件体积**：天龙八部的 characters.json 约 4000+ 行，需要考虑打包体积
   - 解决：Vite 会自动 code-split，按需加载
2. **新旧数据兼容**：部分书籍可能还是旧结构
   - 解决：类型定义用可选字段兼容
3. **性能**：大量数据的表格渲染
   - 解决：TanStack Table 支持虚拟滚动，或使用 `@tanstack/react-virtual`
4. **shadcn/ui 学习成本**：需要了解 Tailwind + Radix UI 的用法
   - 解决：shadcn/ui 文档完善，组件源码在本地可直接修改
5. **中文字体**：需要配置合适的中文字体栈
   - 解决：Tailwind 配置中添加中文字体（PingFang SC、Microsoft YaHei 等）
