## 待修复的问题

### 问题1：verify.js mention coverage 读错路径

**文件**: `scripts/verify.js` 第 286-326 行

**问题**: mention coverage 检查从 `novelDir` 根目录读取 JSON 文件（`path.join(novelDir, file)`），但实际文件在 `data/` 子目录。导致 coverage 报告始终显示 0 条覆盖。

**修复**: 改为从 `novelDir/data/` 读取，与 check-skill-items.js 保持一致。

### 问题2：cross-validate.js 缺少 techniques 和 locations 的验证

**文件**: `scripts/cross-validate.js`

**问题**:
- `allNames` map 不包含 techniques 的 name/alias（第 52 行）
- 不包含 locations 的 alias（第 63 行只加 name，不加 alias）
- 不包含 factions 的 alias（第 60 行只加 name）
- 幻觉检测和重复检测跳过这些实体

**修复**: 在构建 allNames 时，遍历 techniques、locations、factions 的 alias 字段。

### 问题3：verify.js 静默吞掉解析错误

**文件**: `scripts/verify.js` 第 154、168 行

**问题**: `catch {}` 空块静默忽略 JSON 解析错误，不输出任何警告。

**修复**: 改为 `catch (e) { console.warn(...) }`。

### 问题4：check-skill-items.js 缺少复合武功后缀

**文件**: `scripts/check-skill-items.js`

**问题**: `SKILL_SUFFIXES` 只匹配 `法/功/掌/拳/指/腿/步/术/诀/经`，不匹配 `刀法/剑法/棍法/枪法`。导致"五虎断门刀"等明显是武功的条目被误报。

**修复**: 在正则中加入 `刀法|剑法|棍法|枪法|鞭法|杖法|钩法` 等复合后缀。

### 问题5：coverage-gap.js 不包含 techniques

**文件**: `scripts/coverage-gap.js` 第 52-56 行

**问题**: coveredTerms 只包含 characters/factions/locations/skills/items，不包含 techniques。导致 technique 名称在 coverage gap 报告中误报。

**修复**: 加入 techniques 的 name 和 alias。

### 问题6：cross-validate.js SUSPICIOUS_PATTERNS 定义但未使用

**文件**: `scripts/cross-validate.js` 第 343-349 行

**问题**: 定义了可疑模式正则，但没有代码遍历实体进行匹配。

**修复**: 删除未使用的代码，或实现匹配逻辑。
