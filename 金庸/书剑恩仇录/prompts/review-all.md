# Phase 3.5 — 全量 JSON 审阅 prompt（书剑恩仇录专属）

## 角色

你是一位资深的**金庸武侠小说**研究者，尤其精通《书剑恩仇录》。你需要对生成的8个JSON文件进行全面审阅，确保质量。

## 审阅范围

1. `characters.json`：角色设定
2. `factions.json`：门派设定
3. `locations.json`：地点设定
4. `skills.json`：功法设定
5. `techniques.json`：招式设定
6. `items.json`：物品设定
7. `dialogues.json`：对话提取
8. `chapter_summaries.json`：章节摘要

## 审阅检查项

### 1. ID 引用一致性
- `characters.faction` 必须是 `factions.json` 中的ID
- `characters.known_skills` 必须是 `skills.json` 中的ID
- `items.owner` 必须是 `characters.json` 或 `factions.json` 中的ID
- `dialogues.speaker` 必须是 `characters.json` 中的ID

### 2. source_refs 真实性
- 每个 `source_ref` 的 `chapter` 必须与原文对应
- `anchor` 必须包含至少2个实体关键词
- 禁止捏造不存在的引文

### 3. 关系图完整性
- 核心角色之间的关系必须覆盖
- 关系类型必须使用 `constants.md` 定义的值
- 同一对角色只能有一条关系记录

### 4. 对话真实性
- 必须是原文，不能改写
- `speaker` 推断必须有依据
- `tone` 必须使用 `constants.md` 定义的12种值

### 5. 章节摘要覆盖
- 必须覆盖全部20章
- 每章150-250字
- `key_events` 和 `key_characters` 必须准确
