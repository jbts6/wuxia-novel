# Pass 1 — 实体骨架生成 prompt（书剑恩仇录专属）

## 角色

你是一位资深的**金庸武侠小说**研究者兼数据库工程师，尤其精通《书剑恩仇录》。你已经熟读本书，现在需要基于原文写一份完整、准确、无冲突的设定库。

## 输入

1. `manifest.json`：20回章节清单（总字数约150万字）。
2. `mention_summary.json`：高频实体提及汇总。
3. `outline.json`（可选）：实体清单大纲。
4. 原文（已注入上下文）。

## 本书叙事风格

- **语言特色**：典雅白话与文言交融，对话文雅有礼，叙述细腻
- **时代背景**：清乾隆十八年至二十四年间，汉满矛盾激化
- **叙事视角**：第三人称全知视角，多线并进
- **核心主题**：民族大义、兄弟亲情、爱情悲剧、江湖义气

## source_ref 锚定策略

### first_mention 锚点（首次登场）
- 角色：选其第一次出场或被提及的章节
- 门派：选其第一次在江湖上被提及的章节
- 功法：选其第一次施展或被描述的章节

### climax 锚点（高潮冲突）
- 红花会救文泰来（ch6-10）
- 六和塔对峙（ch11-12）
- 回疆大战（ch14-15）
- 张召重伏法（ch19）
- 香香公主之死（ch20）

### resolution 锚点（结局收束）
- 陈家洛与乾隆决裂（ch20）
- 红花会群雄结局（ch20）
- 霍青桐与香香公主的归宿（ch19-20）

## 已知陷阱

### 容易遗漏的角色
1. **天山双鹰**（陈正德、关明梅）：出场不多，但是霍青桐的师父，关键配角
2. **袁士霄**：陈家洛的师父，天池怪侠，后期重要角色
3. **阿凡提**：回疆智者，出场虽少但形象鲜明
4. **福康安**：乾隆宠臣，后期反派
5. **李沅芷**：陆菲青弟子，暗恋余鱼同，贯穿全书

### 容易遗漏的功法
1. **庖丁解牛功**：陈家洛在玉峰所学，后期核心武功
2. **三分剑术**：霍青桐的剑法
3. **柔云剑术**：武当派剑法，陆菲青、张召重都会
4. **铁琵琶手**：焦文期的武功
5. **黑沙掌**：常氏兄弟的武功

### 容易遗漏的物品
1. **可兰经**：回部圣物，被镖局抢走
2. **温玉**：乾隆赠陈家洛，后转赠香香公主
3. **白龙剑**：陆菲青的佩剑
4. **金笛**：余鱼同的兵器
5. **铁胆**：周仲英的暗器

### 关系图常见错误
1. **陈家洛-乾隆**：是亲兄弟，不是堂兄弟
2. **霍青桐-香香公主**：是亲姐妹，都爱陈家洛
3. **陆菲青-张召重**：是师兄弟，不是师徒
4. **余鱼同-骆冰**：余鱼同暗恋骆冰，骆冰是文泰来之妻
5. **于万亭-陈家洛**：于万亭是陈家洛义父，不是亲父

## 输出

5 个 JSON 文件，严格按照 `schemas.md` 和 `constants.md` 的 schema、ID 规则、枚举值：

- `characters.json`
- `factions.json`
- `locations.json`
- `skills.json`
- `techniques.json`

### 生成顺序（重要！）
1. 先生成 `factions.json` 和 `skills.json`
2. 再生成 `characters.json`
3. 最后生成 `locations.json` 和 `techniques.json`

### 硬性约束（保留通用模板全部约束）
- 每个实体必须附 `source_refs: [{chapter, anchor, event_type}]`
- 关系图无冲突、ID 无重复、枚举值合法
- `one_line` 必须反映全书定位（≤40字）
- 禁止英文占位或问号兜底

## 输出格式

每个文件是单个 JSON 数组，用 `JSON.stringify(data, null, 2)` 格式。直接输出 5 个 JSON 块的完整内容，用 `=== characters.json ===` 等分隔符标明文件名。
