# Pass 2 — 细节生成 prompt（书剑恩仇录专属）

## 角色

你是一位资深的**金庸武侠小说**研究者，尤其精通《书剑恩仇录》。你需要基于 Pass 1 生成的实体骨架，补充详细的对话、物品、章节摘要等细节。

## 输入

1. Pass 1 的 5 个 JSON 文件（characters.json、factions.json、locations.json、skills.json、techniques.json）。
2. 原文（已注入上下文）。

## 输出

3 个 JSON 文件：
- `items.json`：武器/道具
- `dialogues.json`：代表性对话
- `chapter_summaries.json`：章节摘要

---

## dialogues 选材指南

### 最值得提取的对话场景

1. **陆菲青训徒**（ch1）：教导李沅芷做人道理，体现师徒情深
2. **铁胆庄对峙**（ch3）：周仲英与官差的冲突，展现江湖义气
3. **陈家洛与乾隆相认**（ch11）：兄弟身世揭秘，全书核心转折
4. **霍青桐与陈家洛**（ch14-16）：草原定情，展现儿女柔情
5. **香香公主与乾隆**（ch20）：宁死不屈，悲剧高潮
6. **张召重伏法**（ch19）：陆菲青殉义，师兄弟恩怨终结
7. **红花会群雄议事**（多章）：体现各人物性格特点

### 说话者推断规则
- **显式标注**："XXX 说道"、"XXX 道"、"XXX 笑道"
- **隐式推断**：对话连续性、场景分析、语气分析
- **无法确定时**：`speaker` 设为 `null`

### 本书对话风格
- **陈家洛**：文雅有礼，引经据典，有书生气
- **乾隆**：威严中带狡诈，帝王口气
- **霍青桐**：爽朗直接，有英气
- **香香公主**：天真纯朴，语气稚嫩
- **文泰来**：豪迈直爽
- **徐天宏**：机智冷静
- **余鱼同**：文采斐然，略带酸气
- **周仲英**：老成持重
- **陆菲青**：沉稳睿智
- **张召重**：阴狠狡猾

## items 焦点

### 核心道具类型
1. **兵器**：白龙剑、凝碧剑、金笛、鸳鸯刀、五行轮
2. **暗器**：芙蓉金针、琵琶钉、飞刀
3. **信物**：可兰经、温玉
4. **秘籍**：（本书较少）

### items.json 必填字段
- `rarity_tier`：凡品/良品/珍品/神品（不能全部设为"未知"）
- `tags`：标签数组（兵器/剑/暗器/信物等）

### 物品稀有度参考
| 物品 | rarity_tier | 理由 |
|------|-------------|------|
| 凝碧剑 | 神品 | 削铁如泥的神兵 |
| 白龙剑 | 珍品 | 武当派名剑 |
| 金笛 | 良品 | 余鱼同的标志性兵器 |
| 可兰经 | 神品 | 回部圣物 |
| 温玉 | 良品 | 乾隆御赐之物 |
| 芙蓉金针 | 良品 | 暗器，陆菲青传授 |

## chapter_summaries 重点

### 情节主线
1. **红花会救文泰来**（ch1-10）：全书前半段的核心冲突
2. **陈家洛身世揭秘**（ch11-12）：与乾隆相认，汉满矛盾激化
3. **回疆征战**（ch13-18）：清廷征讨回部，香香公主出场
4. **最终对决**（ch19-20）：张召重伏法，香香公主自尽

### 情节副线
1. **陆菲青线**：从避祸到殉义，贯穿全书
2. **余鱼同-骆冰-文泰来三角**：情感纠葛
3. **霍青桐-香香公主-陈家洛三角**：爱情悲剧
4. **乾隆与太后的权力斗争**：宫廷暗流

### 摘要要求
- 150-250字
- 覆盖主要情节推进、人物变化、关系转折
- `key_events` 列出2-4个关键事件
- `key_characters` 列出本章核心角色的ID

---

## 硬性约束（保留通用模板全部约束）

### dialogues
- 每章挑 5-15 条最能体现角色性格/关系转折的代表性台词
- 必须原文，不重写
- `tone` 必须使用 `constants.md` 定义的12种值之一

### items
- 只收真正重要的物品（推动剧情/体现人物/武学相关）
- 必须为每个物品分配 `rarity_tier`
- 必须包含 `tags` 字段

### chapter_summaries
- 覆盖全部20章
- `key_characters` 必须使用 `characters.json` 中的ID

---

## 输出格式

每个文件是单个 JSON 数组，用 `JSON.stringify(data, null, 2)` 格式。直接输出 3 个 JSON 块的完整内容，用 `=== items.json ===` 等分隔符标明文件名。
