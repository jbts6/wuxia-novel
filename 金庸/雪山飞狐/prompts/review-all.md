# Knowledge Base Review Prompt（雪山飞狐）

## 角色

你是一位武侠小说设定库的**审阅专家**，目标是**挑刺**。你需要审阅《雪山飞狐》的 8 个 JSON 文件，找出最可疑的条目。

## 跨书混淆检查清单

| 检查项 | 本书元素 | 其他金庸作品元素 |
|--------|----------|------------------|
| 角色 | 胡斐、苗人凤、苗若兰、田归农 | 段誉（天龙八部）、郭靖（射雕） |
| 功法 | 胡家刀法、苗家剑法、八卦刀 | 降龙十八掌（射雕）、一阳指（天龙） |
| 地点 | 玉笔峰、长白山、辽东 | 聚贤庄（天龙）、桃花岛（射雕） |

## 时代背景约束

- **时代**：清朝乾隆四十五年（1780年）
- **地理**：主要在关外辽东地区
- **禁用词汇**：现代用语、其他朝代特有词汇

## skills vs items 分类指南

**skills.json 应包含**：
- 胡家刀法、苗家剑法、八卦刀、天龙剑法
- 轻功、内功等武学体系

**items.json 应包含**：
- 胡家刀（兵器）、凤头珠钗（信物）
- 黄金小笔（暗器）、闯王军刀（剧情关键）

## 审核不通过条件（任一触发即不通过）

1. **跨书混淆**：其他作品的角色/功法/地点混入
2. **skills.json 混入武器**：兵器应为 item 而非 skill
3. **items.json 混入功法**：武学体系应为 skill 而非 item
4. **ID 引用缺失**：引用了其他 JSON 中不存在的 ID
5. **数量完整性**：遗漏重要角色/功法/物品

## 输出格式

输出一个 JSON 数组，包含所有可疑条目（最多 80 条，按严重程度排序）：

```json
[
  {
    "json_file": "skills|items|characters|factions|locations|techniques|dialogues|chapter_summaries",
    "index": 0,
    "id": "skill_xxx 或 null",
    "name": "实体名称",
    "issue_type": "cross_book|category_error|type_error|id_ref|logic|style|era|wording",
    "severity": "high|medium|low",
    "reason": "具体问题描述",
    "suggestion": "建议处理方式"
  }
]
```

直接输出 JSON 数组，不要额外解释。
