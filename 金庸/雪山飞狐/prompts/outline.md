# Phase 1.5 — 大纲生成 prompt（雪山飞狐专属）

## 角色

你是一位资深的**武侠小说**研究者兼数据库工程师，尤其精通金庸作品，对《雪山飞狐》的叙事结构、人物关系和武学体系了然于胸。你已经熟读本书（这是你训练语料里的已知文本），现在需要基于原文写一份完整、准确的实体清单大纲。

## 本书简介

《雪山飞狐》是金庸的经典武侠小说，设定于清朝乾隆四十五年（1780年）。故事以关外长白山玉笔峰为中心，围绕胡一刀与苗人凤两家百年的恩怨情仇展开。全书采用独特的"多声部叙事"手法——宝树、苗若兰、独臂仆人（平阿四）等多位角色分别讲述同一段往事的不同版本，真相层层剥开，充满悬疑与反转。胡斐作为主角，在追寻父仇的过程中逐渐成长，最终与苗人凤在玉笔峰决战。

## 输入

1. `manifest.json`：章节清单（章节号、标题、起止行号、字数）。
2. `mention_summary.json`：高频实体提及汇总（从 mention_index.jsonl 聚合）。

## 硬性约束

### 清单完整性

- **角色**：列出所有有名有姓、有情节作用的角色（约 25-35 人，本书角色相对集中）。包括：
  - 核心主角（3-4 人）：胡斐、苗若兰、胡一刀、苗人凤
  - 重要配角（8-10 人）：宝树、曹云奇、田青文、陶子安、阮士中、刘元鹤、殷吉、田归农等
  - 次要角色（8-12 人）：陶百岁、熊元献、周云阳、范帮主等
  - 龙套角色（5-8 人）：商剑鸣、平阿四等
- **门派/家族**：列出所有门派、帮派、家族等组织（约 8-12 个）。本书以家族势力为主，注意区分：
  - 天龙门（门派，分南北两宗）
  - 胡家、苗家、田家、陶家（家族）
  - 其他帮派组织
- **地点**：列出所有有情节意义的地点（约 10-15 个）。本书地理跨度较大：
  - 辽东/关外（胡一刀故事发生地）
  - 北京（部分情节发生地）
  - 雪山/玉笔峰（主要场景）
  - 其他回忆中出现的地点
- **功法**：列出所有被命名的武学体系（约 10-15 个）。本书武学描写相对内敛，注重实战而非华丽招式。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
  - 对：`辽东大侠胡一刀之子，幼年流落江湖，成年后为父报仇`
  - 错：`在第9章与苗人凤比武`
- `role` 必须基于全书重要性判断，不要因为某章戏份多就标为"核心"。
  - 注意：胡一刀虽在回忆中出场，但其地位等同核心主角
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底：`unknown`、`weapon`、`???`、`?`、`N/A` 都不允许。

### 与 mention_summary 的对齐

- 通读 `mention_summary.json`，确保高频提及的实体（total mentions > 50）都被列入清单。
- 本书高频实体包括：胡斐(230)、曹云奇(228)、宝树(228)、苗若兰(218)、胡一刀(209)、苗人凤(133)、阮士中(132)、陶子安(107)、刘元鹤(105)、田青文(103)
- 低频实体（total mentions < 20）可以酌情省略，但不要遗漏重要的龙套角色。

## 易混淆提醒

- **苗人凤 vs 苗若兰**：父女关系，苗人凤是"打遍天下无敌手"，苗若兰是温雅少女

- **田归农 vs 田青文**：父女关系，田归农是幕后黑手，田青文处境复杂
- **宝树 vs 独臂仆人（平阿四）**：都是叙事者，但立场不同

## 跨书混淆风险

- **胡一刀**：本书人物，与《飞狐外传》胡斐同为系列主角
- **苗人凤**：本书核心人物，在《飞狐外传》中也是重要角色
- **田归农**：本书反派，与天龙门相关
- **赵半山**：红花会三当家，本书中有提及

## 工作流

1. 先通读 `manifest.json`，了解全书章节结构和篇幅（10章，约13万字）。
2. 通读 `mention_summary.json`，建立高频实体基线。
3. 基于全书先验知识，列出 4 个清单：
   - 先列核心角色（胡斐、苗若兰、胡一刀、苗人凤），再扩展到重要、次要、龙套
   - 先列主要家族/门派（天龙门、胡家、苗家、田家），再扩展到次要组织
   - 先列关键地点（雪山、玉笔峰、辽东、北京），再扩展到次要地点
   - 先列核心功法，再扩展到次要功法
4. 对每个实体写一句话定位（one_line），确保反映全书定位。
5. 最后检查：清单完整性、one_line 质量、与 mention_summary 的对齐。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。

## 后续使用

生成的 `outline.json` 将作为 Pass 1 的骨架输入：
- Pass 1 不再需要"列谁"，只需对 outline 中的每个实体填充 details（relationships、personality、source_refs 等）
- LLM 注意力更聚焦，尾部实体的细节质量会提升
- 实体遗漏率降至 0%（outline 已锁定清单）
