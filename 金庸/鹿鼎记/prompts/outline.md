# Phase 1.5 — 大纲生成 prompt（鹿鼎记）

## 角色

你是一位资深的**武侠小说**研究者兼数据库工程师，尤其精通金庸、古龙、梁羽生、黄易四大家的作品。你已经熟读本书（这是你训练语料里的已知文本），现在需要基于原文写一份完整、准确的实体清单大纲。

## 书籍简介

《鹿鼎记》是金庸封笔之作，共50回，约110万字。以清代康熙年间为背景，讲述市井少年韦小宝从扬州丽春院一路混迹至皇宫、天地会、神龙教的传奇经历。小说以反清复明、满汉矛盾为历史大背景，融合政治权谋、江湖义气与爱情喜剧，是金庸作品中最独特的一部——主角不会武功，全凭口才、机智和运气纵横天下。叙事风格典雅白话，语言幽默风趣，主角韦小宝说话市井粗俗，与康熙的帝王气度形成鲜明对比。

## 输入

1. `manifest.json`：章节清单（章节号、标题、起止行号、字数）。
2. `mention_summary.json`：高频实体提及汇总（从 mention_index.jsonl 聚合）。

## 易混淆角色/实体警告

### 跨书混淆风险

1. **少林寺**：本书中的少林寺是清代少林（韦小宝假扮小和尚），不要与《天龙八部》《倚天屠龙记》《笑傲江湖》中的少林寺混淆——虽然同为少林，但时代不同、人物不同、情节不同。

2. **降龙十八掌**：本书不涉及此武功。降龙十八掌是《射雕英雄传》《神雕侠侣》的核心武功。

3. **丐帮**：本书中丐帮不是主要势力，不要与射雕三部曲中的丐帮混淆。

4. **明朝/清朝**：本书设定在清朝康熙年间（约1660-1689年），不要与《碧血剑》（明末）或其他明朝背景作品混淆。

5. **郑成功家族**：本书中的台湾郑家（郑经、郑克塽）是历史真实人物，不要与金庸其他作品混淆。

6. **吴三桂**：本书中的吴三桂是反清叛军首领，不要与《鹿鼎记》以外的文学作品中的吴三桂形象混淆。

### 本书内部易混淆

1. **胖头陀 vs 瘦头陀**：两人都是神龙教高手，胖头陀会化骨绵掌，瘦头陀武功也不弱。两人名字相似但体型相反——胖头陀反而身形瘦小（因练功所致），瘦头陀反而肥胖。注意区分！

2. **郑克塽 vs 郑克爽**：原文写作"郑克塽"（shǎng），不是"郑克爽"。这是台湾郑家的少主。

3. **陈近南 vs 陈禹**：陈近南是天地会总舵主，陈禹是其他角色，不要混淆。

4. **海大富 vs 海大富**：海大富是宫中太监，武功高强，与韦小宝关系复杂（亦敌亦友）。

5. **沐剑屏 vs 沐剑声**：沐剑屏是小郡主，沐剑声是沐王府的公子/掌门。

6. **九难 vs 九难师太**：九难就是九难师太，即明朝长平公主，化名出家。

7. **毛东珠 vs 毛东珠（太后）**：毛东珠是假冒太后的女人，原为神龙教教徒，不要与其他角色混淆。

8. **归辛树 vs 归二娘 vs 归钟**：归家三人——归辛树（丈夫，武功高强）、归二娘（妻子）、归钟（儿子，体弱多病）。不要混淆。

## 硬性约束

### 清单完整性

- **角色**：列出所有有名有姓、有情节作用的角色（约 50-70 人）。包括：
  - 核心主角（3-5 人）
  - 重要配角（15-20 人）
  - 次要角色（20-30 人）
  - 龙套角色（10-15 人）
  - 背景角色（可选，5-10 人）
- **门派**：列出所有门派、帮派、家族、军队、王族、寺院等组织（约 15-25 个）。
- **地点**：列出所有有情节意义的地点（约 20-35 个）。
- **功法**：列出所有被命名的武学体系（约 15-25 个）。注意：本书武功体系不如射雕丰富，很多"武功"其实是道具或机关。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
  - 对：`市井少年，全凭口才机智纵横皇宫与江湖，七妻环绕的传奇人物`
  - 错：`在第2回被海大富带入皇宫`
- `role` 必须基于全书重要性判断，不要因为某章戏份多就标为"核心"。
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底：`unknown`、`weapon`、`???`、`?`、`N/A` 都不允许。

### 与 mention_summary 的对齐

- 通读 `mention_summary.json`，确保高频提及的实体（total mentions > 50）都被列入清单。
- 如果某个高频实体在清单中缺失，说明你可能遗漏了重要角色/地点/功法。
- 低频实体（total mentions < 20）可以酌情省略，但不要遗漏重要的龙套角色。

## 工作流

1. 先通读 `manifest.json`，了解全书章节结构和篇幅。
2. 通读 `mention_summary.json`，建立高频实体基线。
3. 基于全书先验知识，列出 4 个清单：
   - 先列核心角色（3-5 人），再扩展到重要、次要、龙套
   - 先列主要门派（5-10 个），再扩展到次要组织
   - 先列关键地点（10-15 个），再扩展到次要地点
   - 先列核心功法（5-10 个），再扩展到次要功法
4. 对每个实体写一句话定位（one_line），确保反映全书定位。
5. 最后检查：清单完整性、one_line 质量、与 mention_summary 的对齐。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。

## 后续使用

生成的 `outline.json` 将作为 Pass 1 的骨架输入：
- Pass 1 不再需要"列谁"，只需对 outline 中的每个实体填充 details（relationships、personality、source_refs 等）
- LLM 注意力更聚焦，尾部实体的细节质量会提升
- 实体遗漏率降至 0%（outline 已锁定清单）
