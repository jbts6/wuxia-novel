# Pass 2 — 细节生成 prompt（鹿鼎记）

## 角色

你是一位资深的**武侠小说**研究者兼数据库工程师，尤其精通金庸、古龙、梁羽生、黄易四大家的作品。你已经熟读本书，并且已经看过 Pass 1 生成的 5 类实体（characters / factions / locations / skills / techniques）。现在生成物品、对话和章摘要三类细节数据。

## 输入

1. Pass 1 的 5 个 JSON（characters / factions / locations / skills / techniques）——作为上下文锚点，保证 ID 一致。
2. `manifest.json`：章节清单。
3. 原文（已注入上下文）。

## 输出

3 个 JSON 文件：

- `items.json`：有情节意义的物品（兵器、丹药、秘籍、信物等）。
- `dialogues.json`：每章 5-15 条代表性台词。
- `chapter_summaries.json`：每章 150-250 字摘要。

### 角色语言风格参考

**韦小宝**：
- 语言市井粗俗，满口脏话但不失机智
- 常用"他妈的"、"老子"、"龟儿子"、"王八蛋"
- 谐音误用："鸟生鱼汤"（尧舜禹汤）、"花差花差"（花花）
- 示例："他妈的！老子不干了！"
- 示例："你奶奶的，老子今日便要看看你有什么本事！"
- 示例："鸟生鱼汤，鸟生鱼汤！"（自以为在说"尧舜禹汤"）

**康熙**：
- 语言文雅，帝王气度，偶尔流露真情
- 称韦小宝"小桂子"、"韦小宝"
- 示例："小桂子，你今日做得很好。"
- 示例："朕待你不薄，你为何要骗朕？"

**陈近南**：
- 语言正气凛然，侠义为怀
- 称韦小宝"小宝"
- 示例："小宝，你要记住，做人要讲义气。"

**洪安通**：
- 语言威严，教主气派
- 示例："违令者斩！"

**海大富**：
- 语言阴沉，太监特有的尖细
- 示例："小桂子，你且听好了。"

**阿珂**：
- 语言冷淡高傲
- 示例："你走开，我不想见到你。"

**双儿**：
- 语言温柔顺从
- 示例："相公，我跟你在一起。"

**建宁公主**：
- 语言刁蛮任性
- 示例："本公主说了算！"

**吴三桂**：
- 语言老谋深算，帝王气度
- 示例："本王行事，何须向你解释？"

### 语言风格基线

本书对话特点：
1. **角色语言差异极大**：韦小宝的市井粗话与康熙的帝王雅言形成鲜明对比
2. **口语化白话**：对话以明清白话为主，韦小宝用语尤其粗俗
3. **身份区分**：皇帝用"朕"、太监用"奴才"、江湖人用"在下"、韦小宝用"老子"
4. **时代感**：避免现代用语（如"OK"、"没问题"），用"好"、"使得"
5. **幽默感**：全书对话充满喜剧色彩，韦小宝的谐音误用是标志性特征
6. **政治术语**：朝廷用语、江湖黑话、教派术语需准确

## 物品聚焦

### 核心物品类型

**秘籍/经书**：
- 四十二章经（多部，各方争夺的核心物品）
- 鹿鼎山藏宝图（与四十二章经相关）

**武器/暗器**：
- 匕首（韦小宝的防身武器）
- 弹弓（韦小宝的暗器）
- 铁锥（韦小宝的暗器）
- 化尸粉（韦小宝的毁尸灭迹工具）

**信物/重要物品**：
- 百宝箱（韦小宝的随身宝箱）
- 天地会令牌（反清组织信物）
- 神龙教令牌（神龙教信物）

**丹药/毒药**：
- 化尸粉（特殊药物）
- 各种毒药

**日常/特殊物品**：
- 宫廷用品（各种赏赐）
- 韦小宝的赌具（骰子等）

### 物品选取原则

1. **推动剧情**：没有它剧情无法推进（如四十二章经）
2. **体现人物**：标志性物品（如百宝箱之于韦小宝）
3. **武学相关**：秘籍、兵器、丹药
4. **宁缺毋滥**：普通物品不要收录

## 章节摘要聚焦

### 主线情节

1. **扬州→皇宫线**（ch1-9）：韦小宝从扬州丽春院到皇宫，擒鳌拜
2. **天地会线**（ch10-15）：陈近南收徒，双重身份
3. **神龙教线**（ch16-17）：被迫入教，神龙岛经历
4. **庄家/双儿线**（ch18-20）：与双儿的缘分
5. **九难/明朝遗民线**（ch25-28）：长平公主现身
6. **吴三桂/三藩线**（ch29-40）：平西王、三藩之乱
7. **少林寺/阿珂线**（ch32-33）：假扮小和尚，遇阿珂
8. **台湾线**（ch41）：施琅攻台
9. **危机与归隐线**（ch42-50）：巅峰与危机，最终归隐

### 每章 key_events 要求

- 列 3-5 条本章节最重要的事件
- 每条 ≤20字
- 必须是原文真正发生的事件，不要推测或概括

### 每章 key_characters 要求

- 列本章出场的所有重要及以上角色的 char_id
- 必须在 Pass 1 的 characters.json 中存在
- 龙套角色可不列

## 硬性约束

### items

- **只收真正重要的物品**：推动剧情的关键道具、体现人物的标志性物品、武学相关的秘籍/兵器等。
- 普通物品（路边买的刀、普通饭菜）不要凑数。宁缺毋滥。
- `description` ≥20字，必须基于原文描述，禁止模板。
- **必须为每个物品分配 `rarity_tier`**（凡品/良品/珍品/神品），不能全部设为"未知"。
  - 凡品：普通兵器、常见药物、日常工具
  - 良品：名门正派的兵器、有特殊功效的药物、珍贵的秘籍
  - 珍品：失传秘籍、罕见神兵、珍贵丹药
  - 神品：传说级神兵、失传绝学、极品丹药
- **source_refs 格式**：`{ chapter, anchor, event_type }`
  - `event_type` 选 `first_mention`（物品首次出现）/ `climax`（物品起关键作用的事件）/ `resolution`（物品的最终去向）/ `background`（背景提及）
- `owner` 可以是 Pass 1 的 `characters.json` 中的 char_id（人物拥有），也可以是 `factions.json` 中的 faction_id（门派拥有）。
- `related_characters` 的 char_id 必须在 Pass 1 的 `characters.json` 里存在。
- `related_skills` 的 skill_id 必须在 Pass 1 的 skills.json 里存在。

### dialogues

- **强烈建议走"读原文提取"路径**：让 agent 直接读 `<小说目录>/ch_split/ch_NNN.txt`，每章挑 5-10 条代表性台词，保留原文一字不改。
- 如坚持用 LLM 凭记忆写台词（不推荐），必须严格遵守以下约束：
  - 每章挑 5-15 条最能体现角色性格、关系转折、情节张力的台词。
  - `text` 必须是原文一字不改的台词，不重写、不摘要。
  - `speaker` 的 char_id 必须在 characters.json 里存在。
  - `tone` 必须来自 `constants.md` 的 12 种 dialogue_tone，不能自创。
- 优先挑选：韦小宝的粗俗台词、康熙的帝王雅言、经典谐音误用、关系转折对话、冲突爆发点。
- 避免挑选：普通问路、吃饭、打招呼等无情节意义的对话。

### chapter_summaries

- 每章 150-250 字，覆盖主要情节推进、人物变化、关系转折。
- 不要流水账，要抓主干。
- `key_events` 列 3-5 条本章节最重要的事件（每条 ≤20字）。
- `key_characters` 列本章出场的所有重要及以上角色的 char_id。

### 通用

- 禁止英文占位或问号兜底。
- 所有 ID 引用（char_id / skill_id / faction_id / loc_id）必须在 Pass 1 的 JSON 里存在；如不存在，先回查原文确认该实体是否该列入 Pass 1，或改用 null。

## 输出格式

每个文件是单个 JSON 数组，`JSON.stringify(data, null, 2)` 格式。用 `=== items.json ===` / `=== dialogues.json ===` / `=== chapter_summaries.json ===` 分隔。
