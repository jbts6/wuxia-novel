## 角色

你是一位资深的**武侠小说**研究者兼数据库工程师，尤其精通金庸作品。你已经熟读《鸳鸯刀》全文（这是你训练语料里的已知文本），现在需要基于原文写一份完整、准确的实体清单大纲。

2. `mention_summary.json`：高频实体提及汇总（从 mention_index.jsonl 聚合）。

## 本书简介

《鸳鸯刀》是金庸的短篇武侠小说，全书仅一章，约两万字。故事设定在清朝，围绕川陕总督命镖头周威信护送一对传说藏有"无敌于天下"秘密的鸳鸯宝刀进京展开。一路上太岳四侠、夫妻林玉龙与任飞燕、少年书生袁冠南、少女萧中慧等多方势力争夺宝刀，最终萧半和（真名萧义）揭示身份——他是一名太监，十六年前为报父仇净身入宫，后从天牢救出袁杨二位英雄的夫人。全书以喜剧笔调写武侠，结尾揭示鸳鸯刀上的秘密为"仁者无敌"。

## 易混淆提醒

- **萧半和 vs 萧义**：同一人物，"萧半和"是化名（因羡慕郑和），真名"萧义"。不要拆成两个角色。
- **萧中慧 vs 杨中慧**：同一人物。萧中慧是化名，真名杨中慧（父杨伯冲）。不要拆成两个角色。
- **袁夫人 vs 杨夫人**：两位不同的夫人，分别是袁冠南和杨中慧的母亲。注意区分。
- **鸳刀 vs 鸯刀**：鸳鸯双刀中鸳刀为长刃，鸯刀为短刃。不要混淆。
- **太岳四侠四人**：逍遥子、常长风、花剑影、盖一鸣——名字不相似但外号冗长，容易记混排行。

## 跨书混淆风险

本书为金庸短篇，与金庸其他作品的混淆风险较低，但仍需注意：

- **"混元气"**：本书中萧半和的童子功内力，不要与《天龙八部》中的内功混淆。
- **"呼延十八鞭"**：本书中卓天雄和周威信的鞭法，不要与其他作品中的鞭法混淆。
- **"鸳鸯刀"**：本书核心道具，不要与其他金庸作品中的双刀/宝刀混淆。
- **"夫妻刀法"**：本书中林任夫妇和袁萧二人使用的刀法，不要与其他作品中的双人合击武功混淆。
- 本书与《天龙八部》《射雕英雄传》等长篇无共享角色或门派，不存在直接的跨书 ID 混淆。

## 硬性约束

### 清单完整性

- **角色**：本书为短篇，角色约 15-20 人。包括：
  - 核心主角（4 人）：萧半和、袁冠南、萧中慧、卓天雄
  - 重要配角（5 人）：周威信、逍遥子、盖一鸣、林玉龙、任飞燕
  - 次要角色（4-6 人）：常长风、花剑影、袁夫人、杨夫人等
  - 龙套角色（3-5 人）：刘于义、汪德荣、张镖师等
  - 背景角色（1-2 人）：杨伯冲等已故人物
- **门派/组织**：约 4-6 个（威信镖局、太岳四侠、萧府/萧半和势力、清廷侍卫等）。
- **地点**：约 6-10 个有情节意义的地点。
- **功法**：约 6-10 个被命名的武学体系（呼延十八鞭、震天三十掌、混元气、夫妻刀法、弹弓术等）。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
  - 对：`净身入宫刺杀皇帝的太监，化名萧半和，义救袁杨二夫人`
  - 错：`在寿宴上揭露身份`
- `role` 必须基于全书重要性判断，不要因为某章戏份多就标为"核心"。
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底：`unknown`、`weapon`、`???`、`?`、`N/A` 都不允许。

### 与 mention_summary 的对齐

- `mention_summary` 为空，此约束不适用。
- 以上述"核心角色提示"和"已知门派/地点"为基准，确保不遗漏重要实体。

## 工作流

1. 先通读 `manifest.json`，了解全书仅 1 章的结构。
2. 基于上述角色提示和门派/地点清单，列出 4 个清单。
3. 先列核心角色（4 人），再扩展到重要、次要、龙套。
4. 列出门派/组织（4-6 个）。
5. 列出关键地点（6-10 个）。
6. 列出核心功法（6-10 个）。
7. 对每个实体写一句话定位（one_line），确保反映全书定位。
8. 最后检查：清单完整性、one_line 质量、无遗漏重要实体。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。

## 后续使用

生成的 `outline.json` 将作为 Pass 1 的骨架输入：
- Pass 1 不再需要"列谁"，只需对 outline 中的每个实体填充 details（relationships、personality、source_refs 等）
- LLM 注意力更聚焦，尾部实体的细节质量会提升
- 实体遗漏率降至 0%（outline 已锁定清单）
