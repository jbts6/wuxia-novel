# Pass 2 — 细节生成 prompt（《倚天屠龙记》专属）

## 角色

你是一位精通金庸《倚天屠龙记》的武侠小说研究者兼数据库工程师。你已经熟读本书，现在需要基于 Pass 1 的实体骨架，补充细节：items、dialogues、chapter_summaries。

## 输入

1. `manifest.json`：章节清单。
2. `characters.json`、`factions.json`、`locations.json`、`skills.json`、`techniques.json`（Pass 1 生成）。
3. 原文（已注入上下文）。

## 输出

3 个 JSON 文件，严格按照 `schemas.md` 和 `constants.md`：

- `items.json`：武器、道具、信物等实体物品。
- `dialogues.json`：重要对话片段（必须是原文原句）。
- `chapter_summaries.json`：每章摘要，包含 key_events。

## 硬性约束（必须遵守，否则产物不可用）

### dialogues 真实性

- **必须是原文原句**，禁止凭记忆生成或改写。
- 每条 dialogues 必须附 `source_refs`，包含 chapter、line_start、line_end。
- speaker 和 listener 必须使用 `characters.json` 中的 id，不能使用名称。
- tone 必须是 `constants.md` 中的合法值（陈述、疑问、愤怒、激动、悲伤、恳求、嘲讽、调侃、冷酷、恐惧、欣喜、焦急）。

### items 分类

- `type` 必须是 `constants.md` 中的合法值：兵器/暗器/防具/丹药/毒药/信物/秘籍/奇门/坐骑/食物/工具/饰品/异兽。
- `tags` 必须是 `constants.md` 中的合法值：兵器/刀/剑/枪/棍/棒/暗器/奇门兵器/秘籍/内功/外功/轻功/毒功/丹药/解药/毒药/增益/信物/钥匙/线索/剧情关键。
- `rarity_tier` 必须是 `constants.md` 中的合法值：凡品/良品/珍品/神品/未知。

### chapter_summaries 结构

每章必须包含：
- `chapter`：章节号
- `title`：章节标题
- `summary`：章节摘要（50-100字）
- `key_events`：关键事件列表（每项包含 event_type、chapter、anchor、event_type）

## 工作流

1. 通读 Pass 1 生成的 5 个 JSON，了解实体关系。
2. **生成 `items.json`**：
   - 从原文中提取所有有情节意义的物品
   - 为每个物品分配 type、tags、rarity_tier
   - 附 source_refs
3. **生成 `dialogues.json`**：
   - 从原文中提取重要对话
   - 确保是原文原句
   - 附 source_refs，标注 speaker/listener/tone
4. **生成 `chapter_summaries.json`**：
   - 为每章写摘要
   - 提取 key_events
   - 确保事件分配正确

## 输出格式

每个文件是单个 JSON 数组，用 `JSON.stringify(data, null, 2)` 格式。直接输出 3 个 JSON 块的完整内容，用 `=== items.json ===` 等分隔符标明文件名。
