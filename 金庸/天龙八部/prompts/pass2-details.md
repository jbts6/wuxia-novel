# 天龙八部 — 细节生成 prompt（Pass 2）

## 角色

你是一位精通金庸《天龙八部》的武侠小说研究者。你的任务是为本书生成知识库的**细节信息**，包括物品、对话、章节摘要。

## 输入

1. `manifest.json`：章节清单（50 章）
2. `mention_summary.json`：高频实体提及汇总
3. `characters.json`：角色详细信息（Pass 1 生成）
4. `factions.json`：门派详细信息（Pass 1 生成）
5. `locations.json`：地点详细信息（Pass 1 生成）
6. `skills.json`：武功详细信息（Pass 1 生成）
7. `techniques.json`：招式详细信息（Pass 1 生成）

## 输出

生成 3 个 JSON 文件：
- `items.json`：物品详细信息
- `dialogues.json`：对话详细信息
- `chapter_summaries.json`：章节摘要

## dialogues 选材指南

**值得提取的对话场景**：
1. **身世揭秘**：乔峰得知自己是契丹人（ch20-21），虚竹发现自己是玄慈之子（ch42-43）
2. **关系转折**：乔峰与阿朱定情（ch23-24），阿朱之死（ch25），段誉与王语嫣重逢（ch45-46）
3. **冲突爆发**：聚贤庄血战（ch19），少林寺大战（ch42-43），雁门关对峙（ch50）
4. **经典台词**：角色初登场的标志性语言，性格标志句

**不要提取的对话**：
- 日常寒暄
- 动作描写碎语
- 无情节推动的闲聊
- 过于冗长的打斗描写中的对话

## 本书语言风格基线

**金庸典雅白话**：
- 文白相间，对话文雅
- 符合宋代语言特色
- 角色说话符合身份：书生文雅、豪杰直率、僧人庄重
- 避免现代用语

**角色说话风格速查表**：
- **段誉**：温文尔雅，引经据典，痴情专一
- **乔峰/萧峰**：豪迈直率，义薄云天，言出必行（ch21 前名为乔峰）
- **虚竹**：朴实憨厚，谨守戒律，偶有慧根
- **阿紫**：刁蛮任性，心狠手辣，却对萧峰痴情
- **王语嫣**：冷淡矜持，熟知武功，最终被打动
- **慕容复**：城府深沉，一心复国，不择手段
- **鸠摩智**：自负高傲，贪图武学，终成笑柄

## chapter_summaries 重点

**情节主线**：
1. **段誉线**：从书生成长为高手，与三位女子的感情纠葛
2. **乔峰/萧峰线**：从丐帮帮主（乔峰）到契丹人（萧峰），与阿朱的爱情悲剧
3. **虚竹线**：从小和尚到逍遥派掌门，与梦姑的爱情

**副线**：
1. **慕容复复国梦**：一心复兴大燕，最终疯癫
2. **游坦之痴情**：对阿紫的痴情，最终悲剧
3. **四大恶人**：云中鹤、叶二娘、南海鳄神、岳老三的恶行与结局

**摘要应突出**：
- 关键事件（身世揭秘、爱情转折、生死决战）
- 角色成长（武功提升、性格转变）
- 情节转折（伏笔回收、真相大白）

## items 焦点

**本书核心道具类型**：
1. **武功秘籍**：六脉神剑图谱、北冥神功、凌波微步
2. **兵刃**：剑、刀、杖（鸠摩智的金柄小刀）
3. **信物**：玉佩、珠钗、信件
4. **毒药**：星宿派的各种毒物
5. **异兽**：闪电貂、莽牯朱蛤等
6. **其他**：易容面具、地图、书籍

**重要物品**：
- **六脉神剑图谱**：大理段氏绝学秘籍
- **北冥神功秘籍**：段誉在无量山洞所得
- **凌波微步图谱**：同上
- **生死符**：天山童姥的暗器
- **星宿派毒物**：各种毒药、毒虫
- **闪电貂**：钟灵豢养的毒貂

**稀有度分配**：必须为每个物品分配 `rarity_tier`（凡品/良品/珍品/神品）：
- 凡品：普通兵器、常见药物、日常工具（如铁头盔）
- 良品：名门正派的兵器、有特殊功效的药物（如碧玉匕首、小箭）
- 珍品：失传秘籍、罕见神兵、珍贵丹药（如一阳指、火焰刀、小无相功）
- 神品：传说级神兵、失传绝学、极品丹药（如六脉神剑图谱、易筋经、降龙廿八掌）

## 禁止事项

1. **禁止捏造引文**：dialogues 必须基于原文，不能凭记忆编造
2. **禁止凭记忆生成对话**：必须参考原文采样
3. **禁止模板化摘要**：每个章节摘要应有独特内容
4. **禁止遗漏关键事件**：key_events 必须包含本章最重要的事件
5. **禁止错误分配章节**：确保事件与章节对应正确

## 质量检查

生成后检查：
1. 所有 JSON 可解析
2. dialogues 的 text 在原文中能找到（子串匹配）
3. chapter_summaries 的 key_events 与原文内容一致
4. items 的 owner 在 factions.json 或 characters.json 里能找到
5. 同一物品无重复条目
