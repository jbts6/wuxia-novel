## 本书简介

《神雕侠侣》是金庸「射雕三部曲」第二部，全书40回，约98万字。以南宋理宗年间为时代背景，讲述杨过与小龙女的师徒恋情。二人历经磨难——断臂、中毒、绝情谷、十六年分离——终成眷属。杨过从叛逆少年成长为「神雕大侠」，第三次华山论剑获封「西狂」。全书以「情」为核心主题，贯穿李莫愁、武三通、公孙止等人的痴情与执念。

**核心故事线**：
- **杨过成长线**（全书）：从嘉兴孤儿 → 全真教 → 古墓派学艺 → 行走江湖 → 神雕大侠 → 华山论剑「西狂」
- **师徒恋情线**（全书）：杨过与小龙女历经师徒禁恋、礼教大防、断臂、中毒、绝情谷、十六年等待
- **襄阳抗蒙线**（ch20-40）：郭靖守襄阳，蒙古大军逼近，杨过飞石击毙蒙哥汗解围
- **李莫愁复仇线**（ch1-30）：赤练仙子因情生恨，追杀陆家庄，最终葬身火海

**时代背景**：南宋理宗年间，蒙古南侵，襄阳为抗蒙前线

**叙事特色**：金庸典雅白话，文白相间，诗词引用频繁（如开篇引欧阳修〈蝶恋花〉、「问世间，情是何物」），心理描写细腻，对话文雅含蓄，以景衬情

## 硬性约束

### 清单完整性

- **角色**：列出所有有名有姓、有情节作用的角色（约 50-60 人）。包括：
  - 核心主角（3-5 人）
  - 重要配角（10-15 人）
  - 次要角色（15-20 人）
  - 龙套角色（10-15 人）
  - 背景角色（可选，5-10 人）
- **门派**：列出所有门派、帮派、家族、军队、王族、寺院等组织（约 10-15 个）。
- **地点**：列出所有有情节意义的地点（约 20-30 个）。
- **功法**：列出所有被命名的武学体系（约 25-35 个）。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
  - 对：`杨过之妻，古墓派传人，冰清玉洁，十六年后与杨过重逢`
  - 错：`在第32回跳下绝情谷底`
- `role` 必须基于全书重要性判断，不要因为某章戏份多就标为"核心"。
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底：`unknown`、`weapon`、`???`、`?`、`N/A` 都不允许。

### 与 mention_summary 的对齐

- 通读 `mention_summary.json`，确保高频提及的实体（total mentions > 50）都被列入清单。
- 如果某个高频实体在清单中缺失，说明你可能遗漏了重要角色/地点/功法。
- 低频实体（total mentions < 20）可以酌情省略，但不要遗漏重要的龙套角色。

## 工作流

1. 先通读 `manifest.json`，了解全书章节结构和篇幅（40回，约98万字）。
2. 通读 `mention_summary.json`，建立高频实体基线。
3. 基于全书先验知识，列出 4 个清单：
   - 先列核心角色（杨过、小龙女、郭靖、黄蓉、李莫愁），再扩展到重要、次要、龙套
   - 先列主要门派（古墓派、全真教、丐帮、大理段氏、蒙古阵营），再扩展到次要组织
   - 先列关键地点（绝情谷、襄阳、古墓、华山），再扩展到次要地点
   - 先列核心功法（黯然销魂掌、玉女心经、玄铁剑法、玉女素心剑法），再扩展到次要功法
4. 对每个实体写一句话定位（one_line），确保反映全书定位。
5. 最后检查：清单完整性、one_line 质量、与 mention_summary 的对齐。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。

## 后续使用

生成的 `outline.json` 将作为 Pass 1 的骨架输入：
- Pass 1 不再需要"列谁"，只需对 outline 中的每个实体填充 details（relationships、personality、source_refs 等）
- LLM 注意力更聚焦，尾部实体的细节质量会提升
- 实体遗漏率降至 0%（outline 已锁定清单）
