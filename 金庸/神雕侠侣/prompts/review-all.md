# 《神雕侠侣》Knowledge Base Review Prompt

## 角色

你是一位精通**金庸《神雕侠侣》**的武侠小说设定库审阅专家，目标是**挑刺**。你需要审阅本书的 8 个 JSON 文件，找出最可疑的条目。

## 输入

1. `characters.json`：角色数组
2. `factions.json`：门派/组织数组
3. `locations.json`：地点数组
4. `skills.json`：功法/武学体系数组
5. `techniques.json`：招式数组
6. `items.json`：物品/道具数组
7. `dialogues.json`：对话数组
8. `chapter_summaries.json`：章节摘要数组

## 跨书混淆检查清单（神雕侠侣专属）

本书与同作者其他作品的具体混淆风险点：

### 与《射雕英雄传》混淆（最严重！两书角色大量重叠）

**角色混淆**：
- **郭靖**：《射雕》主角，《神雕》中为重要配角（北侠），身份从少年英雄变为守城大将
- **黄蓉**：《射雕》女主角，《神雕》中为丐帮帮主，从少女变为中年妇人
- **周伯通**：两书都有，但《神雕》中更老，为中顽童
- **一灯大师**：两书都有，但《神雕》中已出家为僧
- **黄药师**：两书都有，东邪
- **柯镇恶**：两书都有，但《神雕》中更老
- **欧阳锋**：《射雕》反派，《神雕》中为杨过义父，已疯癫

**功法混淆**：
- **降龙十八掌**：两书都有，《射雕》中郭靖所学，《神雕》中郭靖仍用
- **九阴真经**：两书都有，《射雕》中为争夺对象，《神雕》中杨过所学
- **一阳指**：两书都有，《射雕》中一灯大师所用，《神雕》中仍为一灯所用
- **蛤蟆功**：两书都有，《射雕》中欧阳锋所创，《神雕》中杨过所学

**地点混淆**：
- **桃花岛**：两书都有，《射雕》中为黄药师居所，《神雕》中仍为黄药师居所
- **襄阳**：两书都有，《射雕》中为抗金前线，《神雕》中为抗蒙前线
- **华山**：两书都有，《射雕》中第一次华山论剑（背景），《神雕》中第三次华山论剑

### 与《倚天屠龙记》混淆

- **张君宝**：《神雕》末尾少年出场（ch40），《倚天》中为张三丰（武当派创始人）
- **峨嵋派**：《神雕》中郭襄创派伏笔，《倚天》中为重要门派
- **九阳真经**：《倚天》中出现，《神雕》中未出现（不要混淆）
- **郭襄**：《神雕》中为少女，《倚天》中为峨嵋派创始人（但主要在《神雕》）

### 与《天龙八部》混淆

- **一阳指**：两书都有，但使用者不同（《天龙》中为段誉，《神雕》中为一灯）
- **大理段氏**：两书都有，但具体人物不同（《天龙》中为段誉家族，《神雕》中为一灯大师）
- **降龙十八掌**：两书都有，但使用者不同（《天龙》中为萧峰，《神雕》中为郭靖）

### 与《笑傲江湖》混淆

- **独孤九剑**：《笑傲》中为令狐冲所学，《神雕》中未出现（不要混淆）
- **独孤求败**：《神雕》中为玄铁重剑的原主人，《笑傲》中为独孤九剑创始人（不要混淆两人的武功）

## 时代背景约束（神雕侠侣专属）

**朝代**：南宋理宗年间（约 1225-1264 年）

**时代错误用语**：
- "小姐"（宋代称"娘子"或"姑娘"）
- "警察"（宋代没有警察，应称"衙役"或"公差"）
- "汽车"等现代事物
- 使用明清以后的称谓（如"老爷"在宋代不常用）
- 使用现代网络用语

**时代背景要点**：
- 蒙古南侵，襄阳为抗蒙前线
- 道教兴盛（全真教）
- 宋代礼教严格（师徒恋被视为禁忌，杨过与小龙女的恋情遭世俗反对）
- 宋人不论达官贵人或里巷小民，无不以唱词为乐

## 角色说话风格速查表（神雕侠侣专属）

核心角色的说话特点：
- **杨过**：叛逆不羁，机智狡黠，深情专一。少年时期调皮捣蛋，成年后沉稳内敛。常以嬉笑怒骂掩饰真情。
- **小龙女**：冷淡直言，不谙世事，说话简洁直接，不拐弯抹角。情感表达含蓄内敛，有时天真得近乎无情。
- **郭靖**：忠厚老实，说话朴实，不善言辞，但句句发自肺腑。对国家忠诚，对朋友义气。
- **黄蓉**：机智聪慧，说话伶俐，有时尖酸刻薄。对女儿溺爱，对杨过防备。
- **李莫愁**：因爱生恨，说话时而柔情似水，时而狠毒无情。常引用「问世间，情是何物」。
- **郭芙**：骄纵任性，说话直白无礼，缺乏同理心。对杨过有复杂感情。
- **郭襄**：天真烂漫，说话活泼，对杨过充满倾慕。聪明伶俐，有侠义心肠。
- **周伯通**：童心未泯，说话天真烂漫，不拘礼节。爱玩爱闹，不拘小节。
- **一灯大师**：慈悲为怀，说话庄重，带有禅意。出家后更显超脱。
- **黄药师**：孤傲不羁，说话傲慢，才华横溢。对女儿溺爱，对世俗不屑。
- **金轮法王**：自负高傲，说话狂妄，但有武者风范。对武学执着。
- **公孙止**：虚伪阴险，说话表面文雅，内心狠毒。好色贪婪。
- **裘千尺**：刚烈暴躁，说话直白狠辣，对丈夫恨之入骨。

## 审阅范围

逐个审阅以下 JSON 文件，找出质量问题：

---

### 1. characters.json 审阅

检查每个角色：

- **跨书混淆**：角色是否属于本书？是否有其他作品的角色混入？
- **身份矛盾**：角色的 identity、role、faction 是否自洽？
- **关系冲突**：relationships 中是否有矛盾（如同一角色既是恋人又是敌人）？
- **别名真实**：alias 是否在原文中真正出现过？
- **personality 空洞**：traits 是否基于全书行为总结，而非套话？
- **ID 引用**：faction、known_skills、related_skills 是否引用了其他 JSON 中存在的 ID？

---

### 2. factions.json 审阅

检查每个门派/组织：

- **跨书混淆**：门派是否属于本书？
- **类型错误**：type 是否正确（门派/帮派/家族/军队等）？
- **地点关联**：location 是否在 locations.json 中存在？
- **ID 引用**：sub_divisions 中的成员是否引用了正确的 ID？

---

### 3. locations.json 审阅

检查每个地点：

- **跨书混淆**：地点是否属于本书？
- **类型错误**：region 是否正确？
- **情节意义**：one_line 是否反映地点的全书定位？

---

### 4. skills.json 审阅（重点！）

检查每个功法/武学体系：

- **武器混入**：是否将武器（玄铁重剑、君子剑、打狗棒等）误列为功法？
  - **区分标准**：
    - skills.json 应包含：内功、掌法、剑法、刀法、鞭法、轻功等**武学体系**
    - items.json 应包含：玄铁重剑、君子剑、打狗棒、玉蜂金针等**武器/道具**
  - **常见错误**：将"玄铁重剑"（武器）列为 skill，应为 item
- **类型正确**：type 是否为合法的 skill.type 枚举值？
- **招式归属**：techniques 数组中的招式是否属于此功法？
- **修炼者存在**：practitioners 中的角色 ID 是否在 characters.json 中存在？
- **跨书混淆**：功法是否属于本书？（如独孤九剑不应出现在本书）

---

### 5. techniques.json 审阅

检查每个招式：

- **归属正确**：source_skill 是否在 skills.json 中存在？
- **跨 skill 检查**：招式是否被多个 skill 使用？
- **描述真实**：description 是否基于原文？

---

### 6. items.json 审阅（重点！）

检查每个物品/道具：

- **武器错放**：是否有功法被误列为物品？
  - **区分标准**：
    - items.json 应包含：玄铁重剑、君子剑、打狗棒、玉蜂金针等**武器/道具**
    - skills.json 应包含：黯然销魂掌、玄铁剑法、玉女心经等**武学体系**
- **地点错放**：是否有地点被误列为物品？
  - **区分标准**：
    - items.json 应包含：可携带、可转移的**实体物品**
    - locations.json 应包含：固定的**地点/场所/建筑物**
  - **常见错误**：将"古墓"（地点）、"绝情谷"（地点）列为 item
- **功法错放**：是否有功法名称被误列为物品？
  - **区分标准**：
    - items.json 应包含：有实体形态的**秘籍/图谱/经书**
    - skills.json 应包含：口传心授的**武功/功法**
  - **常见错误**：将"黯然销魂掌"、"降龙十八掌"等功法名称列为 item（除非原文明确提到有实体秘籍）
- **类型正确**：type 是否为合法的 item.type 枚举值？
- **稀有度合理**：rarity_tier 是否符合物品在本书中的地位？
- **owner 存在**：owner 是否在 characters.json 或 factions.json 中存在？
- **related_skills 存在**：related_skills 中的 ID 是否在 skills.json 中存在？

---

### 7. dialogues.json 审阅

检查每条对话：

- **跨书混淆**：对话内容是否包含其他武侠作品的元素？
- **说话风格不符**：说话风格是否符合角色性格？
- **时代背景错误**：对话是否符合本书的时代背景？
- **措辞偏差**：对话是否像是 LLM 凭记忆写的，而非原文？
- **情节逻辑错误**：对话是否与上下文情节矛盾？
- **speaker/listener 存在**：speaker 和 listener 是否在 characters.json 中存在？

---

### 8. chapter_summaries.json 审阅

检查每章摘要：

- **情节准确**：summary 是否准确反映该章情节？
- **key_events 真实**：事件是否在原文中真正发生？
- **key_characters 存在**：角色 ID 是否在 characters.json 中存在？
- **章节错位**：是否有章节内容被分配到错误的章节号？

---

## 跨 JSON 一致性检查

### ID 引用完整性

- characters.faction → factions.json 中的 ID
- characters.known_skills / related_skills → skills.json 中的 ID
- items.owner → characters.json 或 factions.json 中的 ID
- items.related_skills → skills.json 中的 ID
- dialogues.speaker / listener → characters.json 中的 ID
- chapter_summaries.key_characters → characters.json 中的 ID

### 关系对称性

- 如果 A 的 relationships 包含 B，B 的 relationships 应包含 A（除非单向关系）

### 幻觉检测

- 是否有 ID 被引用但不存在于对应 JSON 中？
- 是否有名称被误用为 ID？

---

## 输出格式

输出一个 JSON 数组，包含所有可疑条目（最多 80 条，按严重程度排序）：

```json
[
  {
    "json_file": "skills|items|characters|factions|locations|techniques|dialogues|chapter_summaries",
    "index": 0,
    "id": "skill_xxx 或 null",
    "name": "实体名称",
    "issue_type": "cross_book|category_error|type_error|id_ref|logic|style|era|wording",
    "severity": "high|medium|low",
    "reason": "具体问题描述",
    "suggestion": "建议处理方式（删除/重分类/重读原文/人工复核）"
  }
]
```

## 工作流

1. 先通读所有 JSON，建立全书设定基线
2. 逐个 JSON 审阅，按上述标准检查
3. 对可疑条目，记录问题类型和严重程度
4. 进行跨 JSON 一致性检查
5. 按严重程度排序输出（high → medium → low）

## 注意事项

- 不要过度挑刺：只标记真正有问题的条目
- 优先标记跨书混淆、类型错误、ID 引用缺失，这些是硬伤
- 对于"可能措辞偏差"，只标记非常明显的问题
- 如果不确定，宁可不标记，避免假阳性
- **重点检查 skills.json 和 items.json 的分类是否正确**

## 审核不通过条件（任一触发即不通过）

1. **跨书混淆**：其他作品的角色/功法/地点混入（severity: high）
2. **skills.json 混入武器**：玄铁重剑、君子剑、打狗棒等应为 item 而非 skill（severity: high）
3. **items.json 混入地点**：古墓、绝情谷、桃花岛等地点应为 location 而非 item（severity: high）
4. **items.json 混入功法名称**：黯然销魂掌、降龙十八掌等功法名称（无实体秘籍）应为 skill 而非 item（severity: high）
5. **ID 引用缺失**：引用了其他 JSON 中不存在的 ID（severity: high）
6. **数量完整性**：对照原文检查是否有重大遗漏
   - 重要角色遗漏（如推动剧情的关键人物）
   - 核心功法遗漏（如主要武学体系）
   - 关键物品遗漏（如核心道具）
   - 如有遗漏，severity 为 high

## 输出

直接输出 JSON 数组，不要额外解释。
