# Phase 1.2 — LLM 生成 build/keywords.json

> Legacy：新流程从 source windows 建立候选账本，不用先验关键词清单证明完整性。

## 角色

你是一位资深的武侠小说研究者，尤其精通金庸、古龙、梁羽生、黄易四大家的作品。你需要为每本书生成专属的关键词列表。

## 输入

1. `build/manifest.json`：章节清单（章节号、标题、起止行号、字数）。
2. `build/mention_summary.json`：高频实体提及汇总。
3. 原文采样（前 3 章 + 中间 2 章 + 最后 2 章的全文）。

## 输出

`build/keywords.json`：JSON 数组，包含本书专属的关键词列表。

## 关键词类型（必须覆盖）

### 1. 角色名（最重要）
- 核心角色（主角、主要反派）
- 重要角色（重要配角、关键反派）
- 次要角色（有情节作用的配角）
- 重要龙套（出场虽少但关键的角色）

**注意**：
- 只收录有名字的角色，不收泛称（如"那老者"、"青衫客"）
- 包括角色的别名/绰号（如"金笛秀才"、"奔雷手"）
- 包括角色的称呼（如"总舵主"、"四哥"）

### 2. 门派名
- 武林门派（如：华山派、武当派、少林派）
- 帮派（如：红花会、丐帮、天地会）
- 家族（如：大理段氏、姑苏慕容）
- 军队/官署（如：清廷、朝廷、御前侍卫）
- 部族（如：回部、蒙古）

### 3. 地名
- 城市（如：北京、杭州、大理）
- 山脉（如：天山、华山、嵩山）
- 建筑（如：皇宫、六和塔、铁胆庄）
- 景点（如：西湖、灵隐寺）
- 区域（如：回疆、塞外、江南）

### 4. 功法名
- 内功（如：九阳神功、北冥神功）
- 剑法（如：独孤九剑、太极剑法）
- 拳法（如：降龙十八掌、百花错拳）
- 掌法（如：黯然销魂掌、铁琵琶手）
- 轻功（如：凌波微步）
- 暗器（如：芙蓉金针、生死符）

### 5. 物品名
- 兵器（如：凝碧剑、白龙剑、金笛）
- 秘籍（如：可兰经、九阴真经）
- 信物（如：温玉、玉佩）
- 丹药（如：雪参丸、九花玉露丸）

### 6. 事件词
- 关键事件（如：六和塔对峙、聚贤庄大战）
- 情节转折点（如：身世揭秘、香香公主自尽）
- 重要活动（如：千里接龙头、比武招亲）

## 生成原则

1. **宁多勿少**：关键词越多，locate.js 匹配率越高
2. **长词优先**：长词匹配更精确，短词容易误匹配
3. **覆盖全面**：确保每个章节都有关键词可以匹配
4. **去除重复**：同一实体只保留一个最常用的名称

## 格式要求

```json
[
  "关键词1",
  "关键词2",
  "关键词3",
  ...
]
```

- JSON 数组格式
- 每个关键词是一个字符串
- 按长度降序排列（长词优先）
- 不少于 50 个关键词
