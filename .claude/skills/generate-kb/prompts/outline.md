# Phase 1.5 — 大纲生成 prompt

## 角色

你是一位资深的**武侠小说**研究者兼数据库工程师，尤其精通金庸、古龙、梁羽生、黄易四大家的作品。你已经熟读本书（这是你训练语料里的已知文本），现在需要基于原文写一份完整、准确的实体清单大纲。

## 输入

1. `manifest.json`：章节清单（章节号、标题、起止行号、字数）。
2. `mention_summary.json`：高频实体提及汇总（从 mention_index.jsonl 聚合）。

## 输出

1 个 JSON 文件 `outline.json`，包含本书的 4 个清单：

```json
{
  "characters": [
    {
      "name": "中文名",
      "role": "核心 | 重要 | 次要 | 龙套 | 背景",
      "faction": "所属门派/家族（可为 null）",
      "one_line": "一句话定位（≤40字）"
    }
  ],
  "factions": [
    {
      "name": "门派名",
      "type": "武林门派 | 帮派 | 家族 | 军队 | 王族 | 寺院 | 部族 | 官署",
      "one_line": "一句话定位（≤40字）"
    }
  ],
  "locations": [
    {
      "name": "地点名",
      "region": "地区（如中原、大理、西夏等）",
      "one_line": "一句话定位（≤40字）"
    }
  ],
  "skills": [
    {
      "name": "功法名",
      "type": "剑法 | 掌法 | 指法 | 拳法 | 刀法 | 枪法 | 棍法 | 杖法 | 棒法 | 暗器 | 阵法 | 奇门兵器 | 内功 | 轻功 | 毒功 | 音攻 | 点穴",
      "one_line": "一句话定位（≤40字）"
    }
  ]
}
```

## 硬性约束

### 清单完整性

- **角色**：列出所有有名有姓、有情节作用的角色（约 40-50 人）。包括：
  - 核心主角（3-5 人）
  - 重要配角（10-15 人）
  - 次要角色（15-20 人）
  - 龙套角色（10-15 人）
  - 背景角色（可选，5-10 人）
- **门派**：列出所有门派、帮派、家族、军队、王族、寺院等组织（约 10-15 个）。
- **地点**：列出所有有情节意义的地点（约 15-25 个）。
- **功法**：列出所有被命名的武学体系（约 20-30 个）。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
  - 对：`大理段氏世子，后为帝，一生追寻王语嫣与身世之谜`
  - 错：`在第3章被蜂麻倒并被缚`
- `role` 必须基于全书重要性判断，不要因为某章戏份多就标为"核心"。
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底：`unknown`、`weapon`、`???`、`?`、`N/A` 都不允许。

### 与 mention_summary 的对齐

- 通读 `mention_summary.json`，确保高频提及的实体（total mentions > 50）都被列入清单。
- 如果某个高频实体在清单中缺失，说明你可能遗漏了重要角色/地点/功法。
- 低频实体（total mentions < 20）可以酌情省略，但不要遗漏重要的龙套角色。

## 工作流

1. 先通读 `manifest.json`，了解全书章节结构和篇幅。
2. 通读 `mention_summary.json`，建立高频实体基线。
3. 基于全书先验知识，列出 4 个清单：
   - 先列核心角色（3-5 人），再扩展到重要、次要、龙套
   - 先列主要门派（5-10 个），再扩展到次要组织
   - 先列关键地点（10-15 个），再扩展到次要地点
   - 先列核心功法（10-15 个），再扩展到次要功法
4. 对每个实体写一句话定位（one_line），确保反映全书定位。
5. 最后检查：清单完整性、one_line 质量、与 mention_summary 的对齐。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。

## 后续使用

生成的 `outline.json` 将作为 Pass 1 的骨架输入：
- Pass 1 不再需要"列谁"，只需对 outline 中的每个实体填充 details（relationships、personality、source_refs 等）
- LLM 注意力更聚焦，尾部实体的细节质量会提升
- 实体遗漏率降至 0%（outline 已锁定清单）
