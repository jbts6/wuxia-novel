# 对话规则

## Speaker

目标：每句“有引号 + 能确定说话人”的话都提取。短句如“是。”、“好。”、“嗯？”也要提取。

硬规则：
- 有直接标注就直接取：`李寻欢道`、`她冷冷道`、`只听阿飞道`。
- 未命名角色也要建角色 ID：`老丐`、`蒙面人`、`白衣女子` 不能直接丢成 `null`。
- 禁止用脚本猜 speaker；必须由 AI 读上下文判断。
- speaker 最多判断 5 秒；仍无可靠线索才用 `null`。
- 内心独白、旁白引语、江湖传言不是角色对话。

快速判断流程：

1. 找直接标注或自报家门作为锚点。
2. 双人连续对话优先按一人一句交替。
3. `他/她` 回溯最近明确行动者，再用立场校验。
4. 行动与语气匹配：宣战者通常拔剑/进攻，冷静回应者通常是被挑衅方。
5. 无锚点、无上下文、无法判断时，`speaker: null`，继续下一句。

不要把这些当成 speaker 名：`他笑着`、`那人`、`有人`、`众人`、`只听`。它们只能作为线索，需要回溯到具体角色。

## Tone

目标：`tone` 只记录说话语气，不记录动作、神态、位置或时间。

硬规则：
- tone 必须来自 `constants.md` 的 `dialogue_tone`；不能自创。
- 无明显语气时用 `陈述`。
- 动作 + 情绪混合时只取情绪：`拍桌怒喝` -> `愤怒`。
- 纯动作无情绪时用 `陈述`：`站起来道`、`转身道`。

常用映射：

| 原文线索 | tone |
|----------|------|
| `道`、`说道`、无修饰 | 陈述 |
| `问`、`追问`、`反问`、`质问` | 疑问 |
| `怒道`、`吼道`、`暴喝` | 愤怒 |
| `冷冷道`、`淡淡道`、`冷然道` | 淡然 |
| `叹道`、`叹了口气` | 无奈 |
| `柔声道`、`温柔道` | 柔声 / 温柔 |
| `悲声道`、`哭道` | 悲伤 / 悲痛 |
| `大笑道`、`哈哈笑道` | 大笑 / 豪迈 / 狂笑 |
| `冷笑道`、`苦笑道`、`微笑道` | 冷笑 / 苦笑 / 微笑 |
| `颤声道`、`嘶声道`、`沉声道`、`厉声道` | 颤声 / 嘶声 / 沉声 / 厉声 |
| `急声道`、`慌忙道` | 焦急 / 慌张 |
| `讥讽道`、`嘲笑道`、`戏谑道` | 嘲讽 / 调侃 |
| `恳求道`、`哀求道` | 恳求 |
| `正色道`、`认真道` | 严肃 / 认真 |

禁止作为 tone：

- 物理动作：拍桌、拔剑、站起、转身、喝酒。
- 面部神态：皱眉、瞪眼、脸红、流泪、闭目。
- 位置时间：门外、窗前、忽然、半晌、片刻后。
- 复合描写整段：`呆了半晌，长叹一声` 应取 `无奈`，不是整句话。
