# speaker 提取规则（核心，必须严格执行）

**⚠️ 不要过度思考 speaker 识别。** 按以下简化流程快速处理，不要反复推敲：

1. **有直接标注**（如"李寻欢道："）→ 直接取名字，不需要思考
2. **无标注** → 用交替模式（上一条的 speaker 交替）
3. **无法判断** → speaker 留 null，继续下一条

**处理一条对话的时间不应超过 5 秒。** 如果你发现自己在反复推敲某条对话的 speaker，立即跳过并标记为 null。

**未命名角色也要有 speaker。** 即使角色被称为"老丐"、"蒙面人"、"白衣女子"，也要创建 ID（如 `char_lao_gai`）并设为 speaker，不要因为没有真名就设为 null。

---

**禁止使用脚本提取 speaker**，必须由 AI 阅读上下文后判定。脚本无法理解中文复杂句式，会把动作描写误认为说话人。

**禁止因"无法确定"而将 speaker 设为 null**。武侠小说的对话有明确的结构规律，绝大多数 speaker 可以通过上下文推断出来。只有在以下情况才允许 speaker 为 null：
- 旁白/叙述性引语（如"书中写道"、"江湖传言"）
- 真正无法判断且上下文无任何线索的极少数情况

**禁止遗漏对话**。每章中每一句有明确说话人的对话都必须提取，包括：
- 短对话："是。"、"好。"、"不错。"、"请。" — 这些短句同样要提取
- 语气词对话："嗯。"、"啊？"、"哦。" — 只要有引号包裹，就要提取
- 争吵/连续对话：多人快速交替的短句，每句都要单独提取

## 规则一：交替对话模式

武侠小说（尤其古龙风格）的双人对话**几乎总是严格一人一句交替**。这是最强的判断依据：

```
"燕十三？"          → A（走近的人，后文自报家门为高通）
"是你的。"          → B（被问的人，即燕十三）
"你的夺命十三剑，真的天下无敌？" → A（继续追问）
"未必。"            → B（回答关于自己武功的问题）
"我就是高通。"      → A（自报家门）
"我知道。"          → B（表示知晓）
```

**操作方法：** 找到一段对话后，先确定第一个说话人（通过直接标注如"XX道"或自报家门），然后**交替分配**后续句子。

## 规则二：代词回溯

当出现"他笑道"、"他淡淡道"时，"他"指代谁需要回溯上下文：

1. 找到最近一次**明确提到姓名**的句子
2. 判断"他"是否指代那个人
3. 注意：如果两人交替对话，"他"通常指**对方**而非自己

```
燕十三淡淡道："要杀我的人并不止你一个。"
高通冷笑："但第一个是我。"
他缓缓拔出剑。
```
→ "他"指高通（最后明确提到的行动者是高通，且拔剑符合高通的宣战立场）

## 规则三：动作/立场一致性

判断"他"是谁时，结合动作的**立场和语气**：

- 说"我一定要杀了你"的人 → 后续拔剑/进攻动作属于此人
- 说"淡淡道"的人 → 通常是被挑衅的一方（冷静回应）
- 说"冷笑"、"讥诮"的人 → 通常是挑衅方

## 规则四：直接标注优先

以下标注可以直接确定 speaker，无需推断：
- `XXX道：`、`XXX笑道：`、`XXX冷冷道：` → 直接取 XXX
- `XXX自报家门`、`XXX道：'我就是XXX'` → 取自报者
- `只听XXX道：` → 取 XXX

## 常见误判（必须排除）

- ❌ "他笑着"、"他冷笑着"、"他怒道"、"他淡淡道" — 这些是动作/神态描写，不是说话人名字
- ❌ "那人"、"少年"、"老者"、"女子" — 无明确指向的代词/称谓，不能直接作为 speaker（需回溯确认身份）
- ❌ "只听"、"有人"、"众人" — 非角色声音

## 提取流程

1. 先通读整段对话，找到**至少一个明确的姓名标注**（如"燕十三淡淡道"、"我就是高通"）
2. 以此为锚点，用**交替模式**向前后扩展，确定每句的 speaker
3. 对"他"的指代，用**代词回溯 + 立场一致性**双重验证
4. 确认每个 speaker 在 `characters.json` 中存在
5. listener 同理，必须是已知角色或 null

## 示例

```
他的脚步沉稳，却走得很快，停在七尺外，忽然问："燕十三？"
"是的。"
"你的夺命十三剑，真的天下无敌？"
"未必。"
这个人笑了，笑得讥诮而冷酷，道："我就是高通，一剑穿心高通。"
"我知道。"
"是你约我来的？"
"我知道你正在找我。"
"不错，我是在找你，因为我一定要杀了你。"
燕十三淡淡道："要杀我的人并不止你一个。"
```

逐句分析：
| #  | 台词 | speaker | 推断依据 |
|----|------|---------|---------|
| 1  | "燕十三？" | 高通 | "他"走近→问"燕十三？"→后文自报"我就是高通" |
| 2  | "是的。" | 燕十三 | 交替：被问者确认身份 |
| 3  | "你的夺命十三剑…" | 高通 | 交替：问对方的武功 |
| 4  | "未必。" | 燕十三 | 交替：回答关于自己的评价 |
| 5  | "我就是高通。" | 高通 | 直接自报家门 |
| 6  | "我知道。" | 燕十三 | 交替：表示知晓 |
| 7  | "是你约我来的？" | 高通 | 交替：质问对方 |
| 8  | "我知道你正在找我。" | 燕十三 | 交替：解释为何出现 |
| 9  | "不错…我一定要杀了你。" | 高通 | 交替：宣战 |
| 10 | "要杀我的人并不止你一个。" | 燕十三 | 直接标注"燕十三淡淡道" |

# tone 提取规则（新增）

**⚠️ 当前 tone 字段最严重的问题是将动作/叙事描写当作语气。** 以下规则必须严格执行。

## 核心原则

**tone 只记录说话时的情绪语气或说话方式，不记录任何动作、神态、位置、时序。**

## 受控词汇表

tone 的值**必须**从 `constants.md` 的 `dialogue_tone` 枚举（41 个合法值）中选择，**不得自创**。包括以下三类：

| 分类 | 合法值 |
|------|--------|
| 情绪语气 | 平静、愤怒、激动、悲伤、悲痛、得意、恐惧、冷酷、温柔、慌张、担心、痛苦、惊讶、疑问、好奇、犹豫、恳求、嘲讽、调侃、豪迈、无奈、认真、焦急、欣慰、欣喜 |
| 说话方式 | 冷笑、苦笑、轻笑、大笑、微笑、狂笑、喃喃、沉声、厉声、颤声、嘶声、柔声、淡然、严肃、低语、娇声 |
| 兜底 | 陈述（无法判断时使用） |

## 原文到 tone 的映射规则

根据对话前的叙述文字，提取**情绪语气**而非整段描写：

| 原文形式 | 正确 tone | 说明 |
|----------|-----------|------|
| "XX道" / "XX说道" / "XX开口道" | **陈述** | 无情绪修饰，用兜底 |
| "XX笑道" / "XX笑着说" | **得意 / 调侃 / 欣喜 / 平静** | 根据上下文判断笑的性质 |

## 否定清单（什么不能放进 tone）

以下类别**禁止**作为 tone 值，必须从中提取情绪语气或归入"陈述"：

```
❌ 物理动作：拍桌、站起、转身、喝酒、拔剑、走过去、坐下
❌ 面部神态：皱眉、瞪眼、脸红、流泪、闭目、睁大眼睛
❌ 空间位置：门外、窗前、座上、阶下、马背上
❌ 时序词：半晌、忽然、突然、片刻后、过了良久
❌ 复合描述："呆了半晌，长叹一声"
   → 正确：无奈（只取叹气反映的情绪）
❌ "哈哈大笑，站了起来"
   → 正确：大笑（只取笑，不取"站了起来"）
❌ "一骨碌爬起来"
   → 正确：陈述（纯动作，无情绪信息）
```

## 判断流程

1. 找到对话前的叙述文字
2. 从中识别**情绪/语气成分**（而不是第一个动词）
3. 如果情绪词和动作词混合（如"拍桌怒喝"），只取情绪词（"愤怒"）
4. 如果叙述全是动作无情绪（如"长身而起"），用"陈述"
5. 如果完全无修饰（直接"XX道"或引号开始），用"陈述"
6. 最终值必须属于 `dialogue_tone` 枚举

## 示例

```
原文：小鱼儿拍桌怒吼道："你们给我闭嘴！"
正确 tone：愤怒
错误 tone：拍桌怒吼、拍桌、吼

原文：燕南天叹了口气，黯然道："你走吧。"
正确 tone：无奈
错误 tone：黯然、叹了口气

原文：苏樱柔声笑道："你又何必这么认真呢？"
正确 tone：调侃（根据上下文）或 柔声
错误 tone：柔声笑道（整段）

原文：花无缺道："我已决定了。"
正确 tone：陈述
错误 tone：（留空或自创）
```
