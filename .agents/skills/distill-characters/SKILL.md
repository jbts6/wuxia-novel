# 提炼角色

从 `<小说目录>/characters.json` 提炼出干净的角色数据：去除非人物条目、合并重复角色、分配重要性等级、修正身份描述。

## 核心原则

**像编纂人物图鉴一样筛选。** 有名有姓或有专属称号的真实人物才配列入图鉴。群体、物件、甲乙丙丁编号不配。

## 删除标准

满足以下任一条件的角色直接删除：

| 类型 | 示例 |
|------|------|
| 群体角色 | 大理百姓、围攻群豪、治瘴百姓、丐帮帮众 |
| 甲乙丙丁编号 | 太医甲、太医乙、大宋军官甲、乡人甲 |
| 石雕/玉像/物件 | 神仙姊姊（石雕） |
| 纯职务称呼（无姓氏） | 内堂侍卫、小婢、店主人、伏击大汉 |
| 无名描述性角色 | 碧衣女子、使钩女子、围庄汉子 |
| 重复别名条目 | 秋水妹（李秋水已存在） |

**判断关键**：角色是否有真实姓名或专属称号？「碧衣女子」是描述，不是名字。「南海鳄神」是专属称号，保留。

## 合并标准

同一人物存在多个条目时，合并为一条：

| 信号 | 示例 |
|------|------|
| 同名重复 | 段正淳 ×2、萧峰 ×2 |
| A 的 name = B 的 alias | 钟夫人 ↔ 甘宝宝、马夫人 ↔ 康敏 |
| 别名指向同一人 | 慕容先生 = 慕容复、吐蕃和尚 = 鸠摩智 |
| 同一身份不同称呼 | 吴长风 = 吴长老 = 红脸长老 |

**合并规则**：
- 保留信息最丰富的条目作为主条目
- 被合并条目的 alias、relationships、source_refs、traits 合并到主条目
- 全局引用修复：所有角色的 relationships 中引用被合并 ID 的，改为主条目 ID

**常见陷阱**：
- 「带头大哥」是玄慈，不是段正淳（段正淳只是被误认）
- 合并前确认两人确实是同一人，不要仅凭名字相似就合并

## 重要性等级（五级制）

| 等级 | 说明 | 数量指引 |
|------|------|----------|
| 核心 | 故事主线驱动者，三大主角 + 主要反派 | 5–8 人 |
| 重要 | 关键配角，有独立故事线或重大剧情影响 | 15–25 人 |
| 次要 | 有名有姓，有明确戏份和关系网 | 40–80 人 |
| 龙套 | 有名有姓，但戏份极少 | 50–100 人 |
| 背景 | 极少出场，仅有名字的边缘角色 | 其余 |

**分级信号**：

| 信号 | 加分 |
|------|------|
| relationships ≥ 6 | +3 |
| relationships ≥ 4 | +2 |
| source_refs ≥ 10 | +3 |
| source_refs ≥ 5 | +2 |
| 出现章节 ≥ 10 | +3 |
| 出现章节 ≥ 5 | +2 |
| known_skills ≥ 2 | +2 |
| identity 含重要头衔（长老/掌门/帮主/谷主/方丈/大师/国师/公子/王爷/公主/前辈/神医/恶人/舵主） | +2 |
| 四大恶人成员 | +3 |

- 总分 ≥ 6 → 次要
- 总分 ≥ 3 → 龙套
- 总分 < 3 → 背景

**手动指定**（不依赖算法）：
- 核心：故事三大主角 + 最主要反派
- 重要：有独立感情线的角色、关键长辈、重大事件核心人物

## 身份修正

identity 字段应描述角色的**具体身份**，而非泛称：

| 差 | 好 |
|------|------|
| 江湖人物 | 丐帮四长老之一，红脸，手使鬼头刀 |
| 逍遥派前辈 | 逍遥派掌门，聪辩先生师父 |
| 外寺高僧 | 五台山清凉寺方丈 |

检查规则：identity 为「江湖人物」「武林人物」「XX人物」的，必须补充具体内容。

## 执行

1. 确认 `<小说目录>/characters.json` 存在且 `<小说目录>/archive/characters.json` 不存在
2. 将 `<小说目录>/characters.json` 复制到 `<小说目录>/archive/characters.json`
3. 读取 `archive/characters.json`
4. **删除**：按删除标准逐条判断，删除非人物条目
5. **合并**：按合并标准识别重复角色，执行合并 + 全局引用修复
6. **分级**：先手动指定核心/重要角色，再用信号算法自动分级其余角色
7. **修正身份**：遍历所有角色，identity 为泛称的补充具体内容
8. **人工复核**：输出 `characters_pending.md`，列出分级不确定的角色供用户确认
9. 写入 `<小说目录>/characters.json` + `<小说目录>/characters_summary.md`

## characters_summary.md 格式

```markdown
# 《书名》角色清单

共 N 人（原 M 人，删除 X，合并 Y）

## 分级统计
| 等级 | 数量 |
|------|------|
| 核心 | 7 |
| 重要 | 21 |
| 次要 | 27 |
| 龙套 | 75 |
| 背景 | 227 |

## 核心角色
| 角色 | 身份 |
|------|------|
| 段誉 | 大理段氏青年 |
| ... | ... |

## 重要角色
| 角色 | 身份 |
|------|------|
| 木婉清 | 蒙面黑衣少女 |
| ... | ... |
```

## 产物校验

- JSON 可解析
- 无重复 ID
- 所有 ID 为小写下划线
- `role` 字段值只有 5 种：核心/重要/次要/龙套/背景
- `source_refs` 和 `rag_refs` 原样保留
- 无「江湖人物」「武林人物」等泛称 identity
- 删除率通常 5%–15%
- 合并率通常 3%–8%
