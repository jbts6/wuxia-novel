# 排版规则参考

## 标准化分组工作流

### 流程概述

每章完整处理流程：

```
split-chapter → sub-agent 排版（逐组）→ merge-groups → post-process → validate
```

### 分组原理

`split-chapter.js` 将章节文件按行拆分为均匀的分组，每组输出到 `.ch_groups/{basename}/` 下，命名格式 `{basename}_g{编号}.md`（编号三位补齐）。

例如 `ch_001.md`（45 行，15 行/组）：
```
.ch_groups/ch_001/
├── ch_001_g001.md   # 第 1-15 行
├── ch_001_g002.md   # 第 16-30 行
└── ch_001_g003.md   # 第 31-45 行
```

### 职责边界（关键）

**消除 sub agent 差异的核心原则：sub agent 只做一件事——理解内容、应用排版规则、输出纯文本。除此之外什么都不做。**

| 谁来做 | 职责 | 不允许做的事 |
|--------|------|-------------|
| **主 Agent** | 遍历分组文件 → 读取内容 → 构造提示词 → 调用 sub agent → 拿到输出 → 覆写分组文件 → 合并 → 后处理 → 验证 | 不替 sub agent 做排版决策 |
| **Sub Agent** | 接收内容 → 按固定规则格式化 → 输出排版后文本 | **不碰**文件路径、文件名、目录结构、组号、流程顺序、是否需要拆分章节等任何编排决策 |

### Sub Agent 的完整约束

sub agent 收到的提示词只有两部分：排版规则 + 待排版文本。sub agent 不知道：
- 自己在处理哪一章、哪一组
- 前面有哪些组、后面还有多少组
- 最终输出到哪里
- 整体流程进行到哪一步

### 主 Agent 排版循环

```
for each 分组文件 in .ch_groups/{chapter}/:
   1. 读取分组文件内容
   2. 将内容嵌入排版提示词（见下方模板）
   3. 调用 sub agent，只拿输出文本
   4. 将输出直接覆写到原分组文件
   5. node scripts/progress.js --done <分组文件>
```

**不允许**：在提示词中包含任何关于分组、路径、流程的信息，只给内容和规则。

### 进度追踪

`.ch_groups/progress.json` 由 `split-chapter.js` 自动生成，记录每章每组的完成状态。

```
{
  "ch_001": {
    "totalLines": 145,
    "groups": 10,
    "groupSize": 15,
    "status": {
      "ch_001_g001": "done",
      "ch_001_g002": "pending",
      ...
    }
  }
}
```

| 状态 | 含义 |
|------|------|
| `pending` | 等待排版 |
| `done` | 排版完成 |

主 Agent 在每次 sub agent 排版后**立即**调用 `progress.js --done` 更新进度。合并脚本 `merge-groups.js` 在合并前检查进度，发现未完成分组时发出警告。

### 合并原理

`merge-groups.js` 按文件名排序（`ch_001_g001` → `g002` → `g003`）拼接所有分组内容，输出到 `ch_formatted/`。

### 完整单章示例

```bash
# 1. 拆分
node scripts/split-chapter.js ch_original/ch_001.md 15

# 2. 排版（AI 操作：对 .ch_groups/ch_001/ 下每个分组执行 sub agent）

# 3. 合并
node scripts/merge-groups.js ch_001 ch_formatted/ch_001.md

# 4. 后处理
node scripts/post-process.js ch_formatted/ch_001.md ch_formatted/ch_001.md

# 5. 验证
node scripts/post-process.js --validate ch_formatted/ch_001.md
```

### 批量完整流程

```bash
# 1. 拆分所有章节
node scripts/split-chapter.js --batch ch_original

# 2. 排版（AI 操作：遍历 .ch_groups/ 下每个分组）

# 3. 合并所有章节
node scripts/merge-groups.js --batch .ch_groups ch_formatted

# 4. 批量后处理
node scripts/post-process.js --dir ch_original ch_formatted

# 5. 批量验证
node scripts/post-process.js --validate ch_formatted
```

## Sub Agent 排版提示词

主 Agent 构造提示词时，**只拼接规则 + 内容**，不加任何额外说明：

```
严格按照以下 10 条规则排版这段文字。只输出排版后的文本，不要任何额外内容。

1. 无缩进：段落开头不要空格
2. 对话独立成行：说话标记后空一行，对话内容单独一行
   错误：段誉听到这里，心道："原来这女子是无量剑西宗的。"
   正确：段誉听到这里，心道：
   "原来这女子是无量剑西宗的。"
3. 短对话例外：说话标记字数 + 对话内容字数 < 20 时，合并成一行
   正确：段誉道："不错。"
4. 空行分隔：每个段落/对话之间必须有一个空行
5. 行长度：每行 40-80 字，优先在句号/问号/叹号处分行，**禁止在逗号处截断**（逗号属于句中停顿，不应换行），仅当超过 80 字且无句末标点时才考虑逗号
6. 引号统一使用 "，确保配对正确
7. 保留原文所有文字内容，不改情节
8. 章节标题保持原样
9. 特殊符号（省略号……、破折号——）保持不变
10. 不要添加任何额外文字、说明、标记

--- 排版内容开始 ---

[插入内容]

--- 排版内容结束 ---
```

## 说话标记处理

### 基础标记

`说道` `笑道` `问道` `答道` `叫道` `喊道` `怒道` `叹道` `喝道` `低声道` `高声道` `大声道` `轻声道` `微笑道` `冷冷的道` `淡淡的道` `厉声道` `哽咽道` `颤声道` `沉声道` `喃喃道` `含笑道` `正色道` `接口道` `插口道` `抢着道` `接着道` `续道` `又道` `再道` `心想` `寻思` `暗想` `道`

### 规则

- 说话标记 + 全角冒号后，先空一行，再写对话
- 例外：总长度 < 20 字时合并
- `道` 是最低优先级标记（其他含 `道` 的标记优先匹配）
- 新标记通过后处理脚本自动提取并缓存到 `scripts/speech-markers.json`

## 对话独立成行

```
✅ 正确：
段誉听到这里，心道：
"原来这女子是无量剑西宗的。"

❌ 错误：对话和叙述混在同行的任何情况

段誉听到这里，心道："原来这女子是无量剑西宗的。"
那女子轻轻一笑，柔声道："我也是。" 干光豪大喜。  ← 对话后紧接叙述也不允许
```

**规则 2 是格式一致性最关键的一条**。所有对话必须独立成行。

## 长对话拆分

长对话在自然断句处拆分，每段独立成行，段间空行：

```
干光豪道："咱们是一家人了，我怎么还会瞒你？"

"师父说，许多年之前，那时是我太师父当东宗掌门。"

"玉壁上所显现的剑法之精，我太师父别说生平从所未见。"
```

拆分后每段保持引号完整，不跨段引号。

## 行长度

| 范围 | 处理 |
|------|------|
| < 40 字 | 无需处理 |
| 40-80 字 | 正常范围，无需拆分 |
| 80-100 字 | 建议在句号/问号/叹号处拆分 |
| > 100 字 | 必须在句号/问号/叹号处拆分 |

拆分点优先级：句号 > 问号/叹号 > **禁止在逗号处拆分** > 分号 > 冒号。

**关键原则**：逗号是句中停顿，不应作为换行拆分点。一段叙述文字即使超过 80 字，如果中间没有句号/问号/叹号，也要保持完整一行——这比强行在逗号处截断更符合阅读习惯。允许行长度超过 80 字，只要没有合适的句末标点。

## 段落格式

- 段落开头**不要**空格（一个全角空格都不允许）
- 段落/对话之间空一行
- 连续空行不超过一行（即两个连续的换行符是最大间隔）

## 引号规范

- 统一使用 `"` 作为引号（不推荐使用 `""` 或其他变体）
- 引号必须配对
- 引号内对话独立成行

## 验证规则

后处理脚本 `--validate` 模式检查项：

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 段首缩进 | 错误 | 行首全角空格 |
| 连续空行 > 1 行 | 警告 | 超过一个空行 |
| 行尾空格 | 警告 | 行末有多余空格 |
| 行长度 > 60 字 | 警告 | 建议拆分 |
| 行长度 > 80 字 | 错误 | 必须拆分 |
| 引号不配对 | 警告 | 一行内引号未闭合 |
| 对话与叙述混合 | 警告 | 同行既有引号对话和非引号叙述（启发式） |
