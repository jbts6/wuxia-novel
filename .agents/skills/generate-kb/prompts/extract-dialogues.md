# Dialogue Extraction Prompt

你是一个武侠小说对话提取专家。你的任务是从原文中提取所有对话，并通过上下文推断说话者。

## 输入

- 小说名称：`{{novel_name}}`
- 章节范围：第 {{start_chapter}} 章 - 第 {{end_chapter}} 章
- 原文内容：`{{content}}`
- 角色列表：`{{characters}}`

## 任务

1. 阅读原文，理解上下文
2. 提取所有对话（包括直接引语和间接引语）
3. 通过上下文推断每条对话的说话者
4. 输出格式化的对话列表

## 输出格式

```json
[
  {
    "speaker": "char_xxx 或 null",
    "speaker_name": "中文名或称呼",
    "listener": "char_xxx 或 null",
    "text": "对话原文，不重写",
    "tone": "<dialogue_tone>",
    "chapter": 1,
    "line_start": 42,
    "line_end": 45
  }
]
```

## 说话者推断规则

### 显式标注
- 直接标注："XXX 说道"、"XXX 道"、"XXX 笑道"
- 间接标注："他说道"、"那人道"

### 隐式推断
- **对话连续性**：如果前一句话是 A 说的，下一句没有标注说话者，通常是 B 说的
- **上下文线索**：通过对话内容推断（如"爹爹"说明是子女在说话）
- **场景分析**：通过场景推断（如两人对峙，通常是交替说话）
- **语气分析**：通过语气推断（如"本王"说明是王爷在说话）

### 无法确定时
- 如果实在无法确定说话者，`speaker` 设为 `null`
- 不要猜测，宁可不填也不要填错

## 对话提取规则

### 必须提取
- 直接引语：用引号标注的对话
- 间接引语："他说道"、"她问道"等

### 不需要提取
- 内心独白：段誉心想、萧峰暗想
- 旁白叙述：作者的描述性文字
- 重复对话：同一句话重复出现

### 对话边界
- 一句话被拆分成多条时，合并为一条
- 多句话合并成一条时，拆分成多条

## dialogue_tone 枚举

只能使用以下 12 种值，不能自创：

- 陈述（含平静、淡然、低语、严肃、沉声、无奈、苦笑）
- 疑问（含好奇）
- 愤怒（含厉声）
- 激动（含惊讶）
- 悲伤（含悲痛、痛苦、颤声、嘶声）
- 恳求
- 嘲讽（含冷笑）
- 调侃
- 冷酷
- 恐惧
- 欣喜（含微笑、轻笑、大笑、狂笑、欣慰、得意）
- 焦急（含慌张、担心）

## 示例

### 输入
```

段誉搔头道："我..."
```

## 质量要求

1. **真实性**：对话必须是原文，不能重写
2. **完整性**：提取所有对话，不遗漏
3. **准确性**：说话者推断必须准确
4. **代表性**：每章提取 5-15 条最能体现角色性格/关系转折的代表性台词

## 输出要求

1. 输出完整的 JSON 数组
2. 确保 JSON 格式正确，可以直接 parse
3. 每条对话必须有 `speaker`、`speaker_name`、`text`、`chapter` 字段
4. 如果无法确定说话者，`speaker` 设为 `null`
