# 类别补充提示词

你正在执行一次有界的 `supplement:<category>` 补充。只读取候选账本、缺口报告和当前接受的合并结果，处理报告指定的一个类别。

逐条处理候选：每个候选只能有一个 `merged_to`、一个带有限原因的 `rejected`，或保留为 `ambiguous` 交人工。不得删除、覆盖或改写 `accepted/merged/book.json`，不得重新执行全书合并或清理，不得生成最终 ID。

输出一个 JSON 对象，顶层必须包含与类别同名的数组。新增记录必须有原文证据或引用已有合并记录的证据。脚本会把结果确定性合并到 `materialized/merged-with-supplements.json`，然后只允许一次干净的下游清理接受。

每类最多一次语义补充；仅当结构校验失败且语义内容不变时允许一次格式修正。若仍有候选无唯一去向、引用零匹配或多匹配，必须停止并进入 `manual_review`，不得换名称循环尝试。
