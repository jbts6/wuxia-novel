# 逐章提取

一次读取 2–3 个完整章节，但为每章分别输出一个 JSON 草稿。先读取 `schemas.md`，再遵守本提示。

对每章提取人物、事件、重要物品、功法、正式命名招式、门派/势力、地点、代表性对白和章节摘要。事实只能来自当前章节原文。功法和正式招式偏高召回；优先保留经典桥段、鲜明人物、秘籍、剧情关键物、高级药毒和神兵利器。

排除普通日用品、无特殊性的随身武器、描述词伪实体、泛称人物/组织和普通动作。不要创建“动作描写”类别。对白从属于事件，同一事件最多一条。

每条事实写当前章 `source_refs`，其中 `chapter` 与简短原文 `text` 必填，行号可省略。不要生成最终 ID；使用 `local_key`、名称和 `event_local_key` 建立章内关系。

输出必须是纯 JSON，每章一个对象，字段与 `schemas.md` 完全一致。即使某类为空也保留空数组。
