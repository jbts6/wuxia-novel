# 类别定向补漏提示词

你正在执行一次有界的 `recall:<category>` 补漏。原文仍是唯一事实来源。

先读取缺口报告中列出的章节和相关已接受事件，只处理报告指定的一个类别。不要重读全书，不要修改任何 `accepted/` 文件，不要生成最终 ID，也不要为了数量区间补造条目。

输出一个 JSON 对象，顶层必须包含与类别同名的数组，例如 `items` 或 `dialogues`。每条记录必须带原文 `source_refs`；对白必须关联已有事件临时键。若确实没有合格记录，输出空数组，并提供持久化复核：

```json
{
  "items": [],
  "none_found": {
    "chapters": [1, 2, 3],
    "conclusion": "none_found",
    "reason": "列出的章节只有普通日用品，没有剧情作用或特殊能力的物品。"
  }
}
```

每类只允许一次语义生成；只有结构或格式错误时，才允许一次内容语义不变的修正。完成后由脚本把结果投影到 `materialized/candidates.json`，不要把补漏结果并入旧接受文件。
