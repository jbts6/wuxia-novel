# 全书一次性清理草稿

读取 `schemas.md`、已接受的 `merged/book.json` 和 `pre_clean_quantity.json`。只执行这一轮语义清理，输出一个 `stage: "cleaned"` 的 JSON 对象；之后不得因数量再次返工。

- 纠正武器、物品、功法和招式的错类；删除描述词、泛称、无名普通动作及无原文依据的实体。
- 原文明确定名的功法和招式即使低频也保留；“全力一挥”“拍出一掌”等普通动作删除，不新增动作类别。
- 物品只保留 `秘籍`、`剧情关键`、`高级药毒`、`神兵利器`、`其他稀有特殊` 五种准入理由。
- `核心`、`重要`人物保留详细生平、性格、关系和功法信息；`次要`、`龙套`、`背景`人物只留简短定位与必要关联。
- `ambiguities` 必须为空；无法消除的问题应停止提交并转人工处理，不能猜测。
- 对白从属于事件，每个事件最多一条短对白。
- 数量报告仅提示异常。可做一次有原文依据的补漏或删冗，但禁止为进入区间而凑数；写入 `quantity_review: { "consumed": true, "explanations": [...] }`。
- 用 `game_material_candidates` 标记战斗系统原型、经典剧情桥段、角色原型/彩蛋、标志性物品、门派与世界观素材，但只引用类别和原著名称。
- 禁止生成或修补最终 `id`、`*_id`、`*_ids`。

只输出 JSON，不附带解释文字。
