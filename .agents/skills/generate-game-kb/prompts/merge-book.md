# 全书合并草稿

读取 `schemas.md`、全部已接受章节 JSON，以及脚本提供的规范化名称分组。只执行一次全书合并，输出一个 `stage: "merged"` 的 JSON 对象。

- 合并同一实体的章节候选、别名和重复引用；同名异人必须保留为不同实体。
- 跨章事件保留全部相关章节，可保留不连续章节，不拆成多个重复事件。
- 每条对白用 `event_key` 关联合并后的事件本地键；一个事件最多一条对白。
- 每章恰好输出一个 `chapter_summaries` 条目。
- 无法唯一判断的同名或边界问题写入 `ambiguities`，不要猜测。
- 只使用 `local_key`、`canonical_name`、`aliases` 和名称型引用；禁止生成 `id`、`*_id`、`*_ids`。
- 本阶段不按数量凑条目，也不加入原文之外的游戏设定。

只输出 JSON，不附带解释文字。
