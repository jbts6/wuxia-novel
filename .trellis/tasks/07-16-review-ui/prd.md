# 知识库审核页面

## Goal

为知识库构建后的人工审核提供便捷的 UI 界面，让用户能够快速扫描和剔除低质量数据，避免直接编辑大量 JSON/YAML 文件。

## Background

当前知识库构建后，数据以 YAML/JSON 文件形式存储，包含人物、武功、物品、章节摘要等实体。用户需要人工审核这些数据，剔除低质量条目。目前只能直接编辑文件，效率低下。

## Requirements

### 核心功能

1. **数据展示**：卡片视图，每个实体显示名称和简介
2. **筛选功能**：按实体类型筛选（人物、武功、物品）
3. **标记功能**：二元标记（保留/删除）
4. **批量操作**：批量删除所有标记为"删除"的数据
5. **数据格式**：读取和写入 YAML 文件
6. **备份还原**：支持从备份文件还原数据

### 交互设计

1. **卡片视图**：
   - 显示实体名称（突出显示）
   - 显示简介（one_line 或 identity）
   - 点击可展开查看详情
   - 颜色/图标标记状态

2. **标记方式**：
   - 点击卡片或按钮标记为"删除"
   - 再次点击取消标记
   - 标记为删除的卡片用红色/删除线标识

3. **批量操作**：
   - "全部删除"按钮，确认后删除所有标记的数据
   - 自动备份原文件

### 数据流

1. 读取书籍目录下的 YAML 文件（如 `data/characters.yaml`）
2. 解析为实体列表
3. 用户审核标记
4. 保存时写回 YAML 文件（备份原文件）

## Acceptance Criteria

- [ ] 能读取指定书籍目录下的 YAML 文件（characters.yaml、skills.yaml、items.yaml）
- [ ] 卡片视图正确显示实体名称和简介
- [ ] 能按类型筛选实体（人物、武功、物品）
- [ ] 能标记实体为"删除"
- [ ] 能批量删除标记的实体
- [ ] 删除前自动备份原文件
- [ ] 修改后正确写回 YAML 文件
- [ ] 能从备份文件还原数据

## Out of Scope

- 详细的实体编辑功能（只做审核标记）
- 复杂的搜索/过滤功能（保持简单）
- 用户权限管理
- 多用户协作

## Technical Notes

- 使用现有 dashboard 项目（React + Vite + Tailwind + shadcn/ui）
- 使用 js-yaml 库解析 YAML
- 数据存储在书籍目录的 `data/` 文件夹下
- 备份文件命名：`原文件名.backup.时间戳.yaml`

## Notes

- 章节摘要不需要审核（数据量大，通常是自动生成的）
- 不需要撤销功能，依赖备份文件手动恢复
- 备份文件命名：`原文件名.backup.时间戳.yaml`
