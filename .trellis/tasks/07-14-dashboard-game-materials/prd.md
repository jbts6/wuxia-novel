# Dashboard 游戏素材与事件接入

## Goal

让 Dashboard 在不破坏旧八类知识库浏览能力的前提下，展示可选的第九类事件数据和 `generate-game-kb` 生成的游戏素材索引，为武侠游戏的武学、剧情、人物彩蛋、标志性物品及世界观设计提供可检索、可追溯的入口。

## Background

- 当前 Dashboard 以八个 `data/*.json` 文件作为可浏览门槛；服务端 `readBookData()`、前端 `RawNovelData`、`NovelData` 和标准化链路均未消费 `events.json`。
- `reports/game_materials.json` 位于报告目录，不在当前单书数据接口的读取范围内。
- `generate-game-kb` 的既有契约规定：`events.json` 是兼容新增的第九类数据；游戏素材报告是九类数据的 ID 索引，不是第十类事实实体。
- 当前《飞狐外传》样本包含 20 个事件和 44 条游戏素材，覆盖五种素材类型；44 个 `source_id` 均能解析到现有人物、功法、招式、物品、势力、地点或事件 ID，未发现缺失或重复 ID。

## Requirements

- **R1 — 旧数据兼容：** 原八类文件继续作为 Dashboard 唯一的必需浏览文件。缺少 `events.json` 或 `reports/game_materials.json` 的旧书仍须保持原有可浏览状态、计数和页面行为。
- **R2 — 可选扩展读取：** 单书只读 API 可以返回合法的 `events.json` 和 `reports/game_materials.json`；缺失、空文件或格式错误不得使原八类数据失去可浏览性，但格式错误必须形成可见警告，不能被静默伪装成合法空数据。
- **R3 — 游戏素材入口：** 单书 Dashboard 提供独立“游戏素材”页面，并在书籍概览提供精简摘要入口。摘要至少展示总数和五种素材类型的数量。
- **R4 — 素材浏览：** 游戏素材页面支持按“战斗系统原型”“经典剧情桥段”“角色原型/彩蛋”“标志性物品”“门派与世界观素材”筛选，并展示 `relevance`、`suggested_use` 和 `reason`。
- **R5 — 原始实体追溯：** 每条素材通过 `source_id` 解析到原始实体，不复制或改写原著事实。人物、功法、招式、物品、势力和地点跳转至对应实体详情；事件跳转至章回区域中的“关键事件”视图。
- **R6 — 事件展示：** “章回”页面增加“章节摘要 / 关键事件”两个视图。事件至少展示名称、重要性、过程、结果、参与者、地点及章节证据；跨章事件保留全部章节引用。
- **R7 — 只读边界：** Dashboard 只读取和投影知识库文件，不生成、修正、回写或自动轮询任何知识库数据。
- **R8 — 一致的降级行为：** 可选扩展缺失、无数据、格式错误和悬空 `source_id` 必须有彼此可区分的 UI 状态；单条无法解析的素材不得导致整个页面崩溃。
- **R9 — 首版范围：** 游戏素材仅在当前单书中浏览，不进入跨书 `/browse` 全局索引；对话、人物、物品、势力、地点等既有浏览行为和八类文件契约保持不变。
- **R10 — 稳定入口：** 单书侧栏始终显示“游戏素材”。报告缺失时显示“本书尚未生成游戏素材”，合法空报告显示“0 条游戏素材”，报告损坏时显示明确的读取错误；不得通过隐藏入口混淆这三种状态。
- **R11 — 招式入口：** “武功阁”扩展为“功法 / 招式”双视图，招式支持独立列表、详情和 URL 深链接；既有功法链接在不带新参数时继续打开原功法视图。

## Acceptance Criteria

- [ ] **AC1（R1、R7）：** 仅有旧八类 JSON 的书籍仍能通过 `/api/library/status` 和 `/api/library/book-data` 加载，既有页面、计数和只读行为无回归。
- [ ] **AC2（R1、R2）：** 缺失可选文件不会改变八类 `dataCompleteness.required`、`browseable` 或原有实体计数；同时具有九类 JSON 和游戏素材报告的书籍能够取得扩展数据，前端标准化后不丢失字段。
- [ ] **AC3（R2、R8）：** 格式错误的可选文件产生书籍级警告但不阻塞原八类页面，且不会被显示为“0 条素材”。
- [ ] **AC4（R8、R10）：** 任意可浏览书籍都能进入“游戏素材”页面；缺失、合法空报告、损坏报告和扩展接口不可用分别呈现“尚未生成”“0 条”、文件错误和暂时不可用状态。
- [ ] **AC5（R3）：** 书籍概览显示游戏素材总数和五类分布，并能进入独立游戏素材页面。
- [ ] **AC6（R4）：** 游戏素材页面可以组合使用素材类型和重要度筛选，44 条《飞狐外传》样本均可浏览。
- [ ] **AC7（R5、R8）：** 《飞狐外传》44 个素材 `source_id` 全部解析成功，其中 6 条事件素材能够打开对应事件；人为构造的缺失引用只影响对应卡片，页面显示“来源不可解析”并禁止错误跳转。
- [ ] **AC8（R5、R11）：** 《飞狐外传》4 条招式素材能够切换至武功阁的招式视图并打开对应招式详情；既有 `skills?detail=<skill-id>` 链接无回归。
- [ ] **AC9（R6）：** 章回页面能够在章节摘要和 20 个关键事件之间切换，跨章 `source_refs` 均可访问。
- [ ] **AC10（R9）：** `/browse` 不请求、不索引也不展示事件或游戏素材，既有对话和实体浏览行为无回归。
- [ ] **AC11（R3、R4、R6、R11）：** 页面、筛选器、标签页和实体跳转具备键盘操作、可见焦点和可读的无障碍名称。
- [ ] **AC12（R1–R11）：** Dashboard 的单元、组件、服务端 API、构建及既有桌面 E2E 验证通过。

## Out of Scope

- 修改 `generate-kb` 或 `generate-game-kb` 的生成、校验、安装与防循环逻辑。
- 将游戏素材作为第十类事实 JSON，或把报告内容复制进九类实体记录。
- 首版加入跨书游戏素材搜索、素材编辑、人工标注、导出或写回功能。
- 为事件新增独立的主侧栏页面；事件归入现有“章回”信息架构。
