# Dashboard 知识库工作台实施计划

## 实施原则

- 先完成第一阶段并通过人工验收，再开始第二阶段。
- 所有文件扫描和命令建议均为只读逻辑；禁止引入 `child_process`、文件监听、轮询或写接口。
- 先建立并测试状态契约，再编写工作台 UI，避免界面反向定义数据规则。
- 不覆盖工作区中与本任务无关的现有修改。

## 重点风险文件

- `dashboard/vite.config.ts`：本地 API 与路径安全。
- `dashboard/src/stores/useLibraryStore.ts`：从同步静态数据切换到异步目录。
- `dashboard/src/hooks/useBookData.ts`：路由切换、缓存和失败状态。
- `dashboard/src/stores/useNovelStore.ts`：标准化数据装载和旧页面兼容。
- `dashboard/src/pages/Library.tsx`：首页整体替换。
- `dashboard/src/pages/Characters.tsx`：已知白屏回归。
- `dashboard/src/data/books.ts`、`dashboard/scripts/sync-books.ts`：旧书目链路移除。

## 回滚点

- 扫描器/API 与前端迁移分开提交或保持可独立回退。
- 在新 store 稳定前保留旧单书数据类型，不重写所有实体页面。
- 第一阶段验收前不开始第二阶段搜索聚合，避免扩大故障面。
