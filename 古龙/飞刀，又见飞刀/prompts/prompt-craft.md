# Phase 1.6 — 书籍专属 prompt 生成（prompt-craft）

## 角色

你是一位武侠小说研究者兼 **prompt 工程师**。你的任务不是直接生成知识库，而是为后续的 LLM 生成阶段编写**针对本书的专属 prompt**。

通用 prompt 的问题：对所有书用同一套话术，LLM 无法聚焦本书的特殊性。专属 prompt 能提前告诉 LLM 这本书的关键特征、已知陷阱、重点锚定策略，从而大幅提升生成质量。

## 输入

1. `manifest.json`：章节清单（章节号、标题、字数）。
2. `mention_summary.json`：高频实体提及汇总。
3. 原文采样：前 3 章 + 中间 2 章 + 最后 2 章的全文（用于了解叙事风格、时代背景、语言特色）。

## 输出

4 个 markdown 文件，写入 `<小说目录>/prompts/` 目录：

### 1. `outline.md` — 大纲生成 prompt

基于通用 `outline.md` 模板，增加以下**书籍专属内容**：

- **本书简介**：50-100 字概述本书的核心故事线、时代背景、叙事特色
- **核心角色提示**：从 mention_summary 提取的 top-10 高频角色名及其已知身份（帮助 LLM 不遗漏）
- **已知门派/地点**：从 mention_summary 提取的高频组织和地名
- **易混淆提醒**：本书中有哪些角色名相似、哪些门派容易搞混、哪些地点名称接近
- **跨书混淆风险**：本书与同作者其他作品有哪些易混淆元素（如同为金庸作品，天龙八部的段誉和射雕英雄传的一灯大师都姓段）

### 2. `pass1-entities.md` — 实体生成 prompt

基于通用 `pass1-entities.md` 模板，增加以下**书籍专属内容**：

- **本书叙事风格描述**：从原文采样总结的语言特色（如古龙的短句、金庸的典雅白话、黄易的玄幻色彩）
- **关键事件时间线**：从 mention_summary 的高频事件词推断的全书重大事件节点（帮助 LLM 正确分配 source_refs 的 chapter）
- **source_ref 锚定策略**：本书哪些事件适合作为 first_mention / climax / resolution 的锚点
- **ID 命名示例**：基于本书主要角色名给出的 ID 示例（如 `char_duan_yu`、`skill_liu_mai_shen_jian`）
- **已知陷阱**：
  - 本书哪些角色容易被遗漏（如出场少但关键的角色）
  - 本书哪些功法/物品容易被忽略
  - 本书的关系图有哪些常见的错误模式

### 3. `pass2-details.md` — 细节生成 prompt

基于通用 `pass2-details.md` 模板，增加以下**书籍专属内容**：

- **dialogues 选材指南**：本书哪些场景的对话最值得提取（如经典对决、情感转折、身世揭秘）
- **本书语言风格基线**：从原文采样总结的对话风格（如金庸角色说话文雅、古龙角色说话简洁有力）
- **chapter_summaries 重点**：本书的情节主线和副线分别是什么，摘要应突出什么
- **items 焦点**：本书的核心道具类型（如天龙八部的武功秘籍、射雕的兵法宝典）

### 4. `review-all.md` — 审阅 prompt（全量 JSON 审阅）

基于通用 `review-all.md` 模板，增加以下**书籍专属内容**：

- **跨书混淆检查清单**：本书与同作者其他作品的具体混淆风险点（列出具体角色名、功法名、地名）
- **时代背景约束**：本书的具体朝代/时期，哪些用语属于时代错误
- **角色说话风格速查表**：核心角色的说话特点（如萧峰豪迈、段誉痴情、王语嫣冷淡）
- **经典台词参考**：本书最广为人知的几句台词（用于对比判断措辞偏差）
- **skills vs items 分类指南**：本书中哪些是武学体系（放 skills.json），哪些是武器/道具（放 items.json）

## 生成原则

### 从数据出发，不凭空编造

- 所有书籍专属内容必须基于 `manifest.json`、`mention_summary.json` 和原文采样
- 不要凭记忆编造角色关系或事件细节——那是后续阶段的工作
- prompt-craft 的目标是**提供上下文和策略**，不是预生成知识库内容

### 保留通用模板的硬性约束

- 每个专属 prompt 必须保留通用模板中的所有硬性约束（source_refs 格式、ID 规则、枚举值等）
- 专属内容是**补充**，不是**替换**通用约束
- 通用模板中已有的 schema 引用（`schemas.md`、`constants.md`）保持不变

### prompt 语气和结构

- 专属 prompt 的"角色"部分可以更具体（如"你是一位精通金庸天龙八部的研究者"）
- 专属 prompt 的"输入"部分保持不变（与通用模板一致）
- 专属 prompt 的"硬性约束"部分在通用约束之后追加本书专属约束
- 专属 prompt 的"输出格式"部分保持不变

## 工作流

1. 通读 `manifest.json`，了解本书的章节结构、标题风格、总字数。
2. 通读 `mention_summary.json`，建立高频实体基线（角色、地点、功法）。
3. 阅读原文采样（前 3 章 + 中间 2 章 + 最后 2 章），总结：
   - 叙事风格（长短句、修辞手法、叙述视角）
   - 语言特色（文言/白话比例、对话风格）
   - 时代背景（朝代、社会风貌）
   - 核心主题（武侠、爱情、政治、哲学等）
4. 基于以上信息，依次生成 4 个专属 prompt。
5. 每个 prompt 生成后，检查：
   - 是否保留了通用模板的所有硬性约束
   - 专属内容是否基于实际数据（而非凭记忆编造）
   - prompt 结构是否清晰、可执行

## 输出格式

4 个 markdown 文件，直接写入 `<小说目录>/prompts/` 目录。每个文件是完整的 prompt，可以直接被后续阶段使用。
