# Pass 2 — 细节生成 prompt（霸王枪专属）

## 角色

你是一位资深的**古龙武侠小说**研究者兼数据库工程师。你已经熟读《霸王枪》，并且已经看过 Pass 1 生成的 5 类实体。现在生成物品、对话和章摘要三类细节数据。

## dialogues 选材指南

本书最值得提取的对话场景：

1. **丁喜与小马的日常互动**（第1-3回）：展现两人性格对比——丁喜的从容幽默 vs 小马的愤怒直率
2. **邓定侯与小马的对决对话**（第3回）：拳头对拳头后的惺惺相惜
3. **丁喜与邓定侯的调查对话**（第5-8回）：推理分析，逐步揭示阴谋
4. **邓定侯与百里长青的对峙**（第13回）：朋友变对手的痛苦对话
5. **伍先生阴谋败露的揭示**（第14回）：真相大白的对话
6. **归东景的马车酒铺场景**（第7回）：古龙式的黑色幽默

## 本书语言风格基线

古龙《霸王枪》的对话特色：
- **极简对话**：大量短句，甚至单字成句（"好。""嗯。""能。"）
- **反问与自答**：角色经常自问自答（"为什么？""因为他……"）
- **口头禅**：小马必带"他妈的"，丁喜喜欢微笑着说话
- **哲理穿插**：对话中常插入人生感悟
- **不说破**：古龙角色经常话说到一半就停住，留白给读者

## chapter_summaries 重点

本书的情节主线和副线：

- **主线**：联营镖局内奸谋杀案——谁是杀害王老爷子的凶手？谁是"伍先生"？
- **副线1**：丁喜的身世之谜——他与百里长青的父子关系
- **副线2**：小马与邓定侯的对手→朋友关系变化
- **副线3**：强盗与镖客的身份对立与和解

摘要应突出：
- 每章的悬疑推进（新线索、新嫌疑、新揭示）
- 角色关系变化（特别是丁喜-邓定侯、邓定侯-百里长青）
- 古龙式的哲学思辨段落

## items 焦点

本书的核心道具类型：
- **兵器**：霸王枪（核心道具）、袖中剑
- **信物/标志**：开花五犬旗（联营镖局标志）
- **财宝**：七十二颗明珠（谭道的寿礼）
- **信件**：六封信（第6回的关键线索）
- **暗器/毒药**：书中出现的暗杀手段

## 输入

1. Pass 1 的 5 个 JSON（characters / factions / locations / skills / techniques）
2. `manifest.json`：14章
3. 原文（已注入上下文）

## 输出

3 个 JSON 文件：items.json、dialogues.json、chapter_summaries.json

## 硬性约束

### items

- 只收真正重要的物品：霸王枪、五犬旗、明珠、六封信、袖中剑等
- `description` ≥20字，基于原文描述
- `source_refs` 格式：`{ chapter, anchor, event_type }`
- `owner` 和 `related_characters` 必须使用 Pass 1 中的 ID

### dialogues

- **必须走"读原文提取"路径**：读 `ch_split/ch_NNN.txt`，每章挑 5-10 条代表性台词
- `text` 必须是原文一字不改
- `speaker` 的 char_id 必须在 characters.json 里存在
- `tone` 必须来自 constants.md 的 12 种 dialogue_tone
- 优先挑选：人物初登场台词、关系转折对话、经典金句、冲突爆发点
- 避免挑选：普通问路、吃饭、打招呼等无情节意义的对话

### chapter_summaries

- 每章 150-250 字，覆盖主要情节推进、人物变化、关系转折
- `key_events` 列 3-5 条本章节最重要的事件（每条 ≤20字）
- `key_characters` 列本章出场的所有重要及以上角色的 char_id

### 通用

- 禁止英文占位或问号兜底
- 所有 ID 引用必须在 Pass 1 的 JSON 里存在

## 输出格式

每个文件是单个 JSON 数组，`JSON.stringify(data, null, 2)` 格式。用 `=== items.json ===` / `=== dialogues.json ===` / `=== chapter_summaries.json ===` 分隔。
