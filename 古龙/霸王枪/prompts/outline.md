## 本书简介

《霸王枪》是古龙"七种武器"系列之一，以"霸王枪"为线索，讲述了一个关于联营镖局内部阴谋的故事。主角丁喜（聪明的丁喜）和好友小马（愤怒的小马）是两个强盗，意外卷入联营镖局的阴谋调查中。全书以悬疑推理为主线，穿插古龙式的哲学思辨和人物刻画。核心主题是"愤怒"——小马的愤怒代表了一种原始的正义力量。

## 硬性约束

### 清单完整性

- **角色**：列出所有有名有姓、有情节作用的角色（约 30-40 人）。包括：
  - 核心主角（2-3 人）：丁喜、小马、邓定侯
  - 重要配角（8-12 人）：百里长青、归东景、姜新、西门胜、王大小姐、苏小波、岳麟、伍先生等
  - 次要角色（10-15 人）
  - 龙套角色（5-10 人）
- **门派**：列出所有门派、帮派、镖局等组织（约 8-12 个）。
- **地点**：列出所有有情节意义的地点（约 10-15 个）。
- **功法**：列出所有被命名的武学体系（约 8-12 个）。

### 字段质量

- `one_line` 必须反映人物/实体的全书定位（≤40字），不是某章事件。
- `role` 必须基于全书重要性判断。
- `faction` 只填明确的所属关系，不确定时填 null。
- 禁止英文占位或问号兜底。

### 与 mention_summary 的对齐

- 高频提及的实体（少林、嵩山、武当等）都必须列入清单。
- 低频实体可以酌情省略，但不要遗漏重要的龙套角色。

## 工作流

1. 先通读 `manifest.json`，了解全书章节结构和篇幅。
2. 通读 `mention_summary.json`，建立高频实体基线。
3. 基于全书先验知识，列出 4 个清单。
4. 对每个实体写一句话定位（one_line），确保反映全书定位。
5. 最后检查：清单完整性、one_line 质量、与 mention_summary 的对齐。

## 输出格式

输出单个 JSON 对象，用 `JSON.stringify(data, null, 2)` 格式。直接输出 JSON 内容，不要额外解释。
